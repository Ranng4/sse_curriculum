# **《软件工程理论与实践》课程设计**

# **实验指导书**

## **一、课程基本信息**

* **课程名称**：软件工程理论与实践课程设计
* **适用对象**：计算机/软件工程相关专业 大二/大三学生
* **总学时**：32学时
* **课程性质**：实践必修
* **前置知识**：软件工程基础、程序设计基础

## **二、课程目标**

1. **掌握传统软件工程流程及其主要方法技术**：需求→设计→实现→测试→交付。
2. **学会用AI提效**：人在回路中（Human-In-the-Loop），使用AI生成用户故事、类图、数据库结构、代码框架、单元测试、功能测试、重构、API文档等。
3. **理解AI边界**：学会审查、调试、重构AI生成代码和文档，避免“盲目信任AI”。
4. **团队协作产出可运行项目**：一个中等规模的基于Web的信息系统。

### **模块2：AI辅助设计（4学时）**

1. AI辅助架构设计和技术选型；
2. 以需求描述作为输入，使用AI提取类及其属性和操作，并迭代优化；
3. 以需求描述作为输入，使用AI生成ER图，并迭代优化；
4. 利用AI从ER图生成数据库设计，并生成数据库创建脚本，并迭代优化；
5. 基于前后端分离原则，使用AI生成后端RESTful API接口文档模板（OpenAPI 3.0规范的YAML文件），并迭代优化；
6. 依据交互场景，使用AI生成前端页面，并迭代优化；
7. 输出和提交：架构和类设计文档architect.md、前端UI文档ui\_design.md、后端接口文档backend\_api.md和数据库设计文档db.md；
8. 在AI使用文档ai.md中记录和AI有意义的交互过程，包括：原始提示词、AI输出摘要、可能存在的问题、迭代优化等；
9. 由小组负责人在工作完成情况文档assign.md中详细记录团队成员具体完成工作项。

### **模块3：AI辅助编码实现（10学时）——核心学时**

1. 搭建前后端框架、实现业务逻辑、数据库连接；
2. **代码生成**：用AI根据注释生成函数（如“// 验证用户权限并返回JWT token”）；
3. **代码解释**：对复杂算法让AI逐行解释；
4. **重构**：对已有代码利用AI进行重构；
5. **异常处理**：粘贴异常栈给AI，要求给出修复方案；
6. **输出和提交**：可运行的项目代码（前端+后端+SQL脚本）；
7. 在AI使用文档ai.md中记录和AI有意义的交互过程，包括：原始提示词、AI输出摘要、可能存在的问题、迭代优化等；
8. 由小组负责人在工作完成情况文档assign.md中详细记录团队成员具体完成工作项。

### **模块4：AI辅助测试与调试（4学时）**

1. 使用示例提示词进行单元测试：“为以下UserService类的register方法生成Pytest单元测试，包含正常、重复用户、空密码三种情况。”，鼓励进行控制结构单元测试；
2. 使用AI辅助生成测试用例，并用接口测试工具（如Postman或Swagger等）对后端接口进行测试；
3. 根据交互场景描述，利用AI辅助生成功能测试用例，并进行功能测试；
4. 用AI辅助定位bug：提供错误日志，让AI分析可能原因。
5. **输出和提交**：测试报告文档test.md。
6. 在AI使用文档ai.md中记录和AI有意义的交互过程，包括：原始提示词、AI输出摘要、可能存在的问题、迭代优化等；
7. 由小组负责人在工作完成情况文档assign.md中详细记录团队成员具体完成工作项。

### **模块5：上线部署与报告撰写（4学时）**

* **上线部署**：要求本地部署，鼓励云上部署。
* **报告撰写**：
  + 输出和提交：软件安装文档install.md、软件使用说明书user\_guid.md；
  + 将前面模块生成的文档进行整合，提交课程设计报告文档（文件名：项目名-负责人-团队成员1-团队成员2.docx，包括前面全部课程设计工作内容和代码附录）。

## **五、考核方式**

|  |  |  |
| --- | --- | --- |
| **项目** | **占比** | **评价要点** |
| 项目功能完成度 | 35% | 功能特性完成效果和程度 |
| AI使用报告 | 25% | AI使用程度和效果，附Prompt示例 + AI生成物截图 + 人工迭代修改过程说明 |
| 代码质量与测试 | 20% | 代码规范程度、测试覆盖程度 |
| 文档完整性 | 20% | 交付文档内容、结构和格式 |

## **六、AI伦理与诚信提醒**

1. **鼓励使用AI**，但必须在代码注释/报告中**明确标注**哪些段落/函数由AI生成。
2. **禁止**：全盘复制未经理解和判断的AI输出，将AI生成的错误或不当结果直接提交。
3. **要求**：每位学生必须能解释自己项目中任意一段AI生成代码的逻辑。


### **题目三、股票基金投资论坛**

##### 1 用户系统

###### 1.1 注册与认证

* **多方式注册**：支持手机号、邮箱、第三方账号（微信/微博）注册
* **分级认证体系**：
  + 基础认证：手机/邮箱验证
  + 实名认证：身份证+人脸识别（可选）
  + 专业认证：上传从业资格、学历证明等（加V标识）
* **投资者适当性评估**：专业用户需完成风险评估问卷

###### 1.2 个人资料管理

* **基本信息**：昵称、头像、个人简介、投资经验标签
* **投资偏好**：关注领域（A股/港股/美股/基金等）、风险偏好
* **成就系统**：发帖数、精华帖、影响力值、荣誉勋章
* **隐私设置**：控制信息可见范围

##### 2 内容系统

###### 2.1 板块分类

* **市场讨论区**：按市场划分（A股、港股、美股、期货等）
* **主题专区**：
  + 价值投资专区
  + 量化投资专区
  + 基金投资专区
  + 新股/新债讨论
  + 宏观策略研讨
* **公司研究专区**：按行业、个股深度讨论
* **问答求助区**：新手提问、投资解惑
* **板块管理**：所有板块可以动态添加、删除和修改

###### 2.2 发帖与互动

* **多样化发帖**：
  + 普通帖子：文字+图片+基础格式
  + 长文分析：富文本编辑器，支持图表插入
  + 投票调研：投资观点调查、市场情绪投票
  + 实时讨论：盘中的简短文讨论（类似微博）
* **内容交互**：
  + 点赞/收藏/转发功能
  + 多级评论系统（支持楼中楼）
  + @提及功能
  + 私信系统（支持文本、图片）
* **附件支持**：上传PDF、Excel等分析报告（需审核）

##### 3 社交与关系系统

###### 3.1 关注与粉丝

* 用户互相关注机制
* 特别关注列表（星标用户）
* 关注动态推送

###### 3.2 群组功能

* 用户自建投资主题群组
* 群组权限管理（公开/私密/审核加入）
* 群内专属讨论和资料分享

##### 4 信息整合系统

###### **4.1 内容聚合**

* **精华推荐**：编辑精选+算法推荐优质内容
* **热榜排行**：每日/每周热门话题、热门股票讨论
* **个性化feed流**：基于用户关注和兴趣的内容推送

###### **4.2 搜索功能**

* 全文搜索：支持关键词、用户、股票代码搜索
* 高级筛选：按时间、热度、精华、市场等维度过滤
* 搜索联想：输入股票代码/名称自动补全

##### **5 管理运营系统**

###### **5.1 内容审核**

* **自动审核**：敏感词过滤、重复内容检测
* **人工审核**：可疑内容队列、用户举报处理
* **合规检查**：防止荐股、操纵市场等违规内容

###### **5.2 用户管理**

* 用户行为监控（发帖频率、内容质量）
* 积分与等级系统
* 违规处理流程：警告、禁言、封号

###### **5.3 数据分析**

* 论坛活跃度统计
* 热门话题分析
* 用户参与度报告
