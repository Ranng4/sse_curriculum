# AI 使用记录（模块1-5）

## 1. 记录说明

本文件记录课程设计过程中与 AI 的有效交互，包括：

1. 原始提示词
2. AI 输出摘要
3. 发现问题
4. 迭代优化

## 2. 模块1：需求分析

### 交互1：需求理解

- 原始提示词：  
  `对于【股票基金投资论坛系统】，如何理解以下功能：注册认证、板块分类、发帖互动、关注粉丝、搜索推荐、内容审核。`
- 输出摘要：  
  AI 将需求拆分为用户系统、内容系统、社交系统、信息整合系统、管理系统。
- 可能问题：  
  初版输出偏“功能罗列”，缺少优先级和可实现范围。
- 迭代优化：  
  增加“课程周期有限，请输出核心/重要/扩展三级优先级”后，形成 `user_stories.md` 的分级结构。

### 交互2：用户故事生成

- 原始提示词：  
  `作为产品经理，为【股票基金投资论坛】生成功能用户故事，格式必须是“作为一名...我想...以便...”`
- 输出摘要：  
  生成 60+ 条用户故事，并覆盖游客/注册用户/管理员多角色。
- 可能问题：  
  部分故事粒度不一致。
- 迭代优化：  
  要求“每条故事只描述一个可测试目标”，最终沉淀到 `docs/user_stories.md`。

### 交互3：交互场景生成

- 原始提示词：  
  `作为软件分析师，为用户故事 US-01~US-20 编写交互场景（前置、基本流程、异常流程、后置）。`
- 输出摘要：  
  形成标准用例模板，可直接用于测试用例设计。
- 可能问题：  
  个别异常流程不贴近实现。
- 迭代优化：  
  加入“结合 REST 接口边界条件”约束，输出更新至 `docs/user_cases.md` 与 `docs/use_cases.md`。

## 3. 模块2：AI辅助设计

### 交互4：架构与技术选型

- 原始提示词：  
  `为课程项目设计可快速迭代的后端架构，要求支持后续替换数据库。`
- 输出摘要：  
  建议分层：API / Service / Repository / Model / Schema。
- 可能问题：  
  初版建议耦合 ORM 过深。
- 迭代优化：  
  改为“仓储接口化 + 内存实现先行”，形成 `docs/architect.md`。

### 交互5：数据库设计与 SQL

- 原始提示词：  
  `根据论坛需求输出 MySQL 表结构，覆盖用户、板块、帖子、评论、互动、关注关系。`
- 输出摘要：  
  给出 ER 关系和建表语句草案。
- 可能问题：  
  草案里缺少联合主键和全文索引。
- 迭代优化：  
  增补 `post_likes/post_favorites/user_follows` 联合主键及帖子全文索引，落地为 `backend/sql/schema.sql`。

### 交互6：OpenAPI 模板

- 原始提示词：  
  `基于前后端分离，生成 OpenAPI 3.0 YAML 文档模板。`
- 输出摘要：  
  生成接口分组与基础 schema。
- 可能问题：  
  部分路径与本项目前缀不一致。
- 迭代优化：  
  统一调整为 `/api/v1/*`，沉淀到 `docs/openapi.yaml`。

## 4. 模块3：AI辅助编码实现

### 交互7：内容系统核心接口实现

- 原始提示词：  
  `基于现有 Flask 分层结构，实现帖子发布/列表/详情、评论、点赞收藏、热榜和搜索联想接口。`
- 输出摘要：  
  生成 `content` 模型、仓储、服务、路由骨架。
- 可能问题：  
  初稿未校验父评论归属关系。
- 迭代优化：  
  在 `ContentService.create_comment` 增加 “parent_comment_id 必须属于同一 post” 校验。

### 交互8：社交关注功能实现

- 原始提示词：  
  `实现用户关注/取关、关注列表、粉丝列表、统计接口，并接入 feed 过滤。`
- 输出摘要：  
  生成 `social` 仓储与服务，并支持 `following_only` 信息流。
- 可能问题：  
  初稿允许关注自己。
- 迭代优化：  
  增加 `cannot follow yourself` 校验。

### 交互9：测试用例生成

- 原始提示词：  
  `为新增接口生成 unittest 用例，覆盖正常流程与关键状态变化。`
- 输出摘要：  
  自动生成内容闭环与社交闭环测试。
- 可能问题：  
  初版手机号冲突导致测试不稳定。
- 迭代优化：  
  引入 `uuid` 生成唯一手机号，测试稳定通过。

## 5. 模块4：AI辅助测试与调试

### 交互10：异常栈分析（numpy/markitdown）

- 原始提示词：  
  `分析 markitdown 执行时 numpy DLL load failed 的原因并给出可执行替代方案。`
- 输出摘要：  
  识别为全局 Python 环境损坏，改用 `uv run --with "markitdown[docx]"` 隔离执行。
- 可能问题：  
  `uv` 默认缓存目录权限受限。
- 迭代优化：  
  使用项目内缓存目录执行，成功转换文档。

### 交互11：接口测试结果分析

- 原始提示词：  
  `根据 unittest 输出，整理可发布的测试报告要点。`
- 输出摘要：  
  提炼通过项与剩余风险（暂无真实三方服务、暂无持久化）。
- 产物：`docs/test.md`。

## 6. 模块5：部署与文档撰写

### 交互12：安装文档与用户手册生成

- 原始提示词：  
  `基于当前项目结构，生成 install.md 和 user_guid.md，要求可直接按步骤操作。`
- 输出摘要：  
  输出安装、启动、常见调用流程与排障建议。
- 产物：`docs/install.md`、`docs/user_guid.md`。

## 7. 人在回路说明

1. AI 负责“初稿生成、结构化整理、样例草拟”。
2. 人工负责“业务边界确认、字段对齐、异常规则补全、测试核验”。
3. 所有关键输出均经过人工复核后入库提交。
