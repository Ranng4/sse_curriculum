# 股票基金投资论坛系统用户故事文档

## 1. 文档说明

本文档基于《软件工程理论与实践》课程设计实验指导书中“股票基金投资论坛”课题要求，对系统功能进行需求理解、细化和整理，并按照用户故事格式进行描述。

用户故事统一采用如下格式：

> 作为一名……我想……以便……

本文档将用户故事按照系统模块进行分类，包括用户系统、内容系统、社交与关系系统、信息整合与搜索系统、管理运营系统等。同时，为便于后续开发安排，本文档将用户故事划分为三个优先级：

| 优先级 | 含义                                                       |
| ------ | ---------------------------------------------------------- |
| 核心   | 课程设计第一阶段应优先完成的功能，是系统能够正常运行的基础 |
| 重要   | 在核心功能完成后尽量实现，可提升系统完整性和用户体验       |
| 扩展   | 难度较高或开发成本较大，可作为后续优化方向写入文档         |

---

## 2. 用户角色说明

股票基金投资论坛系统面向多个角色，不同角色在系统中的需求有所不同。

| 角色     | 角色说明                                                     |
| -------- | ------------------------------------------------------------ |
| 游客     | 未登录用户，可以浏览公开内容和搜索部分帖子                   |
| 注册用户 | 已注册并登录的普通用户，可以发帖、评论、点赞、收藏和关注他人 |
| 专业用户 | 通过专业认证的用户，可以展示认证标识并发布更具专业性的内容   |
| 管理员   | 负责论坛板块维护、内容审核、用户管理、举报处理和数据统计     |

## 3. 用户系统用户故事

用户系统主要用于完成用户注册、登录、身份认证、个人资料管理、投资偏好设置和隐私控制等功能。

| ----- | ------ | ---------------------------------------------------------------------------------------------------------- |

| US-02 | 核心   | 作为一名注册用户，我想使用账号和密码登录系统，以便进入个人账户并使用论坛功能。                             |
| US-03 | 核心   | 作为一名注册用户，我想安全退出当前账号，以便保护自己的账户信息。                                           |
| US-04 | 核心   | 作为一名注册用户，我想修改昵称、头像和个人简介，以便向其他用户展示我的基本信息。                           |
| US-05 | 核心   | 作为一名注册用户，我想设置自己的投资经验标签，以便让别人了解我的投资背景。                                 |
| US-06 | 核心   | 作为一名注册用户，我想选择自己关注的投资领域，例如 A股、港股、美股、基金等，以便系统向我展示更相关的内容。 |
| US-07 | 重要   | 作为一名注册用户，我想设置自己的风险偏好，以便平台能够更准确地了解我的投资倾向。                           |
| US-08 | 重要   | 作为一名专业用户，我想提交从业资格或学历证明进行专业认证，以便获得认证标识并提高个人内容的可信度。         |
| US-09 | 扩展   | 作为一名专业用户，我想完成投资者风险评估问卷，以便平台判断我是否适合发布专业投资分析内容。                 |
| US-10 | 扩展   | 作为一名用户，我想设置个人资料的可见范围，以便保护自己的隐私信息。                                         |
