# 股票基金投资论坛系统交互场景文档

## 1. 文档说明

本文档基于《软件工程理论与实践》课程设计实验指导书中“股票基金投资论坛”课题要求，以及 `user_stories.md` 中整理的用户故事，进一步细化系统主要功能的交互场景。

交互场景用于描述用户在使用系统时，与系统之间发生的具体交互过程。它可以帮助前端同学明确页面流程，帮助后端同学明确接口和业务逻辑，也可以为后续测试用例编写提供依据。

本文档中的交互场景采用如下结构：

| 项目         | 说明                                      |
| ------------ | ----------------------------------------- |
| 场景编号     | 用 UC-xx 表示，便于后续追踪               |
| 场景名称     | 对应一个相对完整的用户操作目标            |
| 参与者       | 触发该场景的主要用户角色                  |
| 关联用户故事 | 与 `user_stories.md` 中的用户故事编号对应 |
| 前置条件     | 场景开始前必须满足的条件                  |
| 基本流程     | 用户与系统正常交互时的步骤                |
| 异常流程     | 可能出现的错误、限制或特殊情况            |
| 后置结果     | 场景完成后系统状态发生的变化              |

---

## 2. 参与者说明

| 参与者   | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| 游客     | 未登录用户，可以浏览公开内容、查看部分帖子和进行搜索         |

## 3. 核心交互场景

### UC-01 用户注册

| 项目         | 内容                                       |
| ------------ | ------------------------------------------ |
| 场景名称     | 用户注册                                   |
| 参与者       | 游客                                       |
| 关联用户故事 | US-01                                      |
| 前置条件     | 游客未登录系统，并进入注册页面             |
| 后置结果     | 系统创建新用户账号，用户可以使用该账号登录 |

#### 基本流程

1. 游客进入论坛首页。
2. 游客点击“注册”按钮。
3. 系统展示注册页面，要求填写手机号或邮箱、密码、确认密码和验证码。
4. 游客填写注册信息并提交。
5. 系统检查手机号或邮箱格式是否正确。
6. 系统检查密码是否符合长度和安全要求。
7. 系统检查手机号或邮箱是否已经被注册。
8. 系统校验验证码是否正确。
9. 校验通过后，系统创建用户账号。
10. 系统提示“注册成功”，并跳转到登录页面或自动登录进入首页。

#### 异常流程

- A1：手机号或邮箱格式错误，系统提示用户重新输入。
- A2：两次输入的密码不一致，系统提示用户修改确认密码。
- A3：手机号或邮箱已被注册，系统提示用户直接登录或更换账号。
- A4：验证码错误或过期，系统提示用户重新获取验证码。

### UC-02 用户登录

| 项目         | 内容                                     |
| ------------ | ---------------------------------------- |
| 场景名称     | 用户登录                                 |
| 参与者       | 注册用户、专业用户、管理员               |
| 关联用户故事 | US-02                                    |
| 前置条件     | 用户已经完成注册，拥有有效账号           |
| 后置结果     | 用户进入系统，系统记录登录状态和用户身份 |

#### 基本流程

1. 用户进入登录页面。
2. 用户输入手机号、邮箱或用户名。
3. 用户输入密码。
4. 用户点击“登录”按钮。
5. 系统校验账号是否存在。
6. 系统校验密码是否正确。
7. 系统判断用户账号状态是否正常。
8. 登录成功后，系统根据用户角色进入对应页面。
9. 普通用户进入论坛首页，管理员进入后台管理首页。

#### 异常流程

- A1：账号不存在，系统提示“账号不存在或未注册”。
- A2：密码错误，系统提示“账号或密码错误”。
- A3：用户账号被禁用或封禁，系统提示无法登录并显示封禁原因。
- A4：用户连续多次登录失败，系统可要求输入验证码。

### UC-03 修改个人资料

| 项目         | 内容                                     |
| ------------ | ---------------------------------------- |
| 场景名称     | 修改个人资料                             |
| 参与者       | 注册用户、专业用户                       |
| 关联用户故事 | US-03、US-04、US-05、US-06               |
| 前置条件     | 用户已登录系统                           |
| 后置结果     | 用户个人资料被更新，个人主页展示最新信息 |

#### 基本流程

1. 用户登录系统。
2. 用户点击头像或“个人中心”。
3. 系统展示个人资料页面。
4. 用户点击“编辑资料”。
5. 用户修改昵称、头像、个人简介、投资经验标签、关注领域和风险偏好。
6. 用户点击“保存”。

8. 校验通过后，系统保存用户资料。

10. 用户返回个人主页，查看更新后的个人信息。

- A1：昵称为空或超过长度限制，系统提示用户修改。
- A2：头像文件格式不支持，系统提示仅支持 jpg、png 等格式。
- A3：个人简介包含敏感词，系统提示用户修改后重新提交。

### UC-04 浏览论坛板块

| 项目         | 内容                           |
| ------------ | ------------------------------ |
| 场景名称     | 浏览论坛板块                   |
| 参与者       | 游客、注册用户、专业用户       |
| 关联用户故事 | US-11、US-12、US-13            |
| 前置条件     | 系统中已经存在论坛板块         |
| 后置结果     | 用户进入目标板块并查看帖子列表 |

#### 基本流程

1. 用户进入论坛首页。
2. 系统展示板块分类列表，例如 A股、港股、美股、基金投资、量化投资、价值投资等。
3. 用户点击感兴趣的板块。
4. 系统进入该板块页面。
5. 系统展示该板块下的帖子列表。
6. 用户可以按照最新发布、最新回复或热度进行排序。
7. 用户点击某个帖子进入帖子详情页。

#### 异常流程

- A1：板块不存在或已被管理员删除，系统提示“该板块不存在”。
- A2：板块暂无帖子，系统提示“当前板块暂无内容”。
- A3：私密板块需要权限访问，系统提示用户登录或申请权限。

### UC-05 发布普通帖子

| 项目         | 内容                                     |
| ------------ | ---------------------------------------- |
| 场景名称     | 发布普通帖子                             |
| 参与者       | 注册用户、专业用户                       |
| 关联用户故事 | US-14、US-15                             |
| 前置条件     | 用户已登录，并进入发帖页面               |
| 后置结果     | 帖子被系统保存，审核通过后在对应板块展示 |

#### 基本流程

1. 用户登录系统。
2. 用户进入某个论坛板块。
3. 用户点击“发布帖子”。
4. 系统展示发帖页面。
5. 用户选择帖子类型为“普通帖子”。
6. 用户填写标题、正文内容，可选择上传图片。
7. 用户点击“发布”。
8. 系统校验标题和正文是否为空。
9. 系统进行敏感词检测和内容合规检查。
10. 如果内容正常，系统保存帖子。
11. 系统提示“发布成功”。
12. 帖子显示在对应板块的帖子列表中。

#### 异常流程

- A1：用户未登录，系统提示先登录后才能发帖。
- A2：标题为空或正文为空，系统提示补充完整。
- A3：内容包含敏感词，系统提示修改后再发布，或将帖子提交至人工审核。
- A4：用户被禁言，系统提示禁言期间不能发帖。
- A5：图片上传失败，系统提示重新上传或删除图片后发布。

### UC-06 发布长文分析

| 项目         | 内容                                     |
| ------------ | ---------------------------------------- |
| 场景名称     | 发布长文分析                             |
| 参与者       | 注册用户、专业用户                       |
| 关联用户故事 | US-21                                    |
| 前置条件     | 用户已登录，并进入长文发布页面           |
| 后置结果     | 长文分析内容被保存并展示，或进入审核队列 |

#### 基本流程

1. 用户点击“发布长文”。
2. 系统展示富文本编辑页面。
3. 用户填写文章标题、正文、相关股票或基金标签。
4. 用户可插入图片、表格或图表说明。
5. 用户点击“预览”。
6. 系统展示文章预览效果。
7. 用户确认无误后点击“发布”。
8. 系统检查内容格式、敏感词和合规风险。
9. 检查通过后，系统保存文章。
10. 系统将文章展示在对应板块或长文专区。

#### 异常流程

- A1：文章内容过短，系统提示长文分析需要达到最低字数。
- A2：文章包含明显荐股、夸大收益等内容，系统提交人工审核。
- A3：插入图片或图表失败，系统提示用户重新上传。

## 6. 关键业务规则整理

| 编号  | 业务规则                                                                   |
| ----- | -------------------------------------------------------------------------- |
| BR-01 | 未登录用户只能浏览公开内容，不能发帖、评论、点赞、收藏、关注和举报。       |
| BR-02 | 被禁言用户不能发帖和评论，但可以浏览公开内容。                             |
| BR-03 | 被封号用户不能登录系统。                                                   |
| BR-04 | 帖子、评论、私信等用户生成内容需要进行敏感词检测。                         |
| BR-05 | 涉及明显违规荐股、虚假宣传、诱导交易等内容，应进入人工审核或被管理员处理。 |
| BR-06 | 用户不能关注自己。                                                         |
| BR-07 | 同一用户对同一帖子或评论只能保留一个点赞状态。                             |
| BR-08 | 同一用户对同一帖子只能保留一个收藏状态。                                   |
| BR-09 | 管理员删除帖子后，该帖子不再对普通用户展示。                               |
| BR-10 | 精华帖应由管理员设置，普通用户不能自行设置。                               |
| BR-11 | 附件上传后应先审核，审核通过后才能公开展示。                               |
| BR-12 | 论坛首页的热门内容可根据浏览量、评论数、点赞数、收藏数等指标计算。         |
