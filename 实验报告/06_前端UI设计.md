# 前端 UI 设计

## 1. 设计目标

1. 面向股票基金讨论场景，突出信息密度、可读性和快速检索。
2. 首页优先展示板块入口、信息流、热榜和搜索。
3. 登录用户能快速完成发帖、评论、点赞、收藏、关注和私信。
4. 管理员能在后台快速完成帖子审核、用户处理和数据查看。

## 2. 页面结构

| 页面 | 文件位置 | 功能 |
| --- | --- | --- |
| 首页 | `frontend/src/views/forum/HomePage.vue` | 论坛入口、推荐内容、热榜入口 |
| 信息流 | `frontend/src/views/forum/FeedPage.vue` | 最新内容和关注流 |
| 分区页 | `frontend/src/views/forum/SectionPage.vue` | 市场、主题、公司研究、问答分区 |
| 板块页 | `frontend/src/views/forum/BoardPage.vue` | 板块帖子列表 |
| 热榜页 | `frontend/src/views/forum/HotRankPage.vue` | 热门帖子排行 |
| 搜索页 | `frontend/src/views/forum/SearchPage.vue` | 关键词和股票代码搜索 |
| 发帖页 | `frontend/src/views/content/CreatePostPage.vue` | 发布普通帖、长文和实时讨论 |
| 详情页 | `frontend/src/views/content/PostDetailPage.vue` | 帖子正文、评论、互动 |
| 登录注册 | `frontend/src/views/auth` | 用户登录和注册 |
| 个人中心 | `frontend/src/views/profile` | 我的资料、编辑资料、适当性评估 |
| 私信 | `frontend/src/views/messages` | 会话列表和聊天详情 |
| 后台管理 | `frontend/src/views/admin` | 统计、帖子审核、用户管理 |

## 3. 组件设计

| 组件 | 职责 |
| --- | --- |

| `PostCard.vue` | 帖子列表卡片，展示标题、摘要、作者、互动数据 |

## 4. 交互规则

1. 未登录用户触发发帖、评论、点赞、收藏、关注、私信时，跳转登录页或展示登录提示。
2. 点赞和收藏采用即时反馈；接口失败时恢复原状态并提示错误。
3. 发帖成功后跳转到帖子详情页。
4. 搜索页支持关键词和股票代码检索，搜索联想用于提高输入效率。
5. 后台操作需要明确反馈处理结果，例如“已标记”“已通过”“已封禁”。

## 5. 视觉规范

- 主色：金融蓝，用于导航、按钮和链接。
- 强调色：增长绿，用于正向指标和成功状态。
- 警示色：橙红色，用于风险提示、审核驳回和封禁操作。
- 布局：桌面端以三栏信息布局为主，移动端收敛为单栏。
- 字体：中文优先使用清晰的无衬线字体，保证长文本阅读体验。

## 6. 响应式方案

| 宽度 | 布局 |
| --- | --- |
| `>=1200px` | 左侧板块入口、中间帖子流、右侧热榜和推荐 |
| `768px~1199px` | 双栏布局，侧栏可折叠 |
| `<768px` | 单栏布局，导航压缩，发帖入口固定在显著位置 |

## 7. AI辅助设计迭代

AI 初稿偏通用论坛页面，没有突出投资论坛的信息密度和搜索需求。人工调整后强调热榜、股票代码搜索、关注流、后台审核和风险提示，使页面结构更贴合股票基金投资交流场景。
