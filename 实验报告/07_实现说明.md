# 编码实现说明

## 1. 后端实现

后端位于 `backend/app`，采用 Flask 分层架构。入口文件为 `backend/app/main.py`，通过 `create_app()` 注册路由和错误处理器，并提供 `/healthz` 健康检查接口。

已实现模块：

1. 用户认证：注册、登录、基础认证、实名/专业认证状态。
2. 个人资料：基础资料、投资偏好、隐私设置、公开资料查看。
3. 适当性评估：风险问卷、提交答案、风险等级结果。
4. 论坛板块：分区聚合、板块列表、板块 CRUD。
5. 内容系统：发帖、更新、详情、列表、评论、楼中楼、点赞、收藏、信息流、热榜、搜索联想。
6. 社交系统：关注、取关、关注列表、粉丝列表、关注统计、关注流过滤。
7. 私信系统：发送消息、会话列表、会话详情、未读数。
8. 管理后台：论坛统计、帖子审核、用户警告/封禁/解封。

## 2. 前端实现

前端位于 `frontend/src`，采用 Vue 3 和 Vite。页面覆盖论坛首页、分区、板块、帖子详情、发布、搜索、热榜、登录注册、个人中心、私信和后台管理。

关键实现：

1. `router/index.js`：管理前端页面路由。
2. `store/auth.js`：维护登录状态和 token。
3. `utils/api.js`：统一封装后端请求。
4. `components/PostCard.vue`、`CommentItem.vue`：支撑核心内容展示。
5. `views/admin`：支撑后台统计、帖子审核和用户管理展示。

## 3. 数据库与脚本

1. 本地运行默认使用 SQLite，支持数据持久化。
2. MySQL 建表脚本位于 `backend/sql/schema.sql`。
3. 测试和演示数据可通过 `backend/seed_data.py` 初始化。

## 4. 关键业务规则

1. 未登录用户不能发帖、评论、点赞、收藏、关注和私信。
2. 用户不能关注自己，也不能给自己发送私信。
3. 点赞、收藏和关注关系使用唯一关系避免重复状态。
4. 父评论必须属于同一帖子，防止跨帖子回复。
5. 后台用户管理支持警告、封禁和解封，帖子审核支持标记和通过。
6. 热榜根据点赞、收藏、评论等互动指标计算。
