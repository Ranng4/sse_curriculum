# 股票基金投资论坛交互场景

## UC-01 用户注册与登录

- 参与者：游客、注册用户
- 关联用户故事：US-01、US-02
- 前置条件：游客未登录，系统可访问
- 后置结果：用户获得访问令牌并进入系统

基本流程：

1. 用户进入注册页，选择手机号或邮箱注册。
2. 用户填写账号、密码、昵称和验证码。
3. 系统校验账号唯一性、密码强度和验证码。
4. 系统创建用户、初始化资料和隐私设置。
5. 用户登录后，系统返回 `access_token`。
6. 前端在后续请求中携带 `Authorization: Bearer <token>`。

异常流程：

1. 账号已存在：返回冲突错误，提示更换账号或直接登录。
2. 密码为空或格式错误：返回参数错误。
3. token 缺失或失效：访问受保护接口时返回 `401`。

## UC-02 修改个人资料与投资偏好

- 参与者：注册用户
- 关联用户故事：US-03、US-04、US-05
- 前置条件：用户已登录
- 后置结果：用户资料、投资偏好或风险测评结果被更新

基本流程：

1. 用户进入个人中心。
2. 用户修改昵称、头像、简介、投资经验标签和关注市场。
3. 系统校验字段长度与枚举值。
4. 用户提交风险评估问卷。
5. 系统计算风险等级并保存评估结果。

异常流程：

1. 昵称为空或过长：系统拒绝保存。
2. 风险问卷答案缺失：系统提示补全。
3. 未登录访问个人接口：系统返回 `401`。

## UC-03 浏览板块与帖子列表

- 参与者：游客、注册用户
- 关联用户故事：US-08、US-21、US-22
- 前置条件：系统存在默认板块
- 后置结果：用户查看板块内容、首页流或热榜

基本流程：

1. 用户进入首页。
2. 系统展示市场讨论、主题专区、公司研究、问答求助四类板块。
3. 用户点击板块进入板块页。
4. 系统按发布时间、热度或筛选条件展示帖子。
5. 用户可进入热榜查看热门帖子。

异常流程：

1. 板块不存在：返回 `404`。
2. 板块下没有帖子：展示空状态。
3. 排序参数非法：返回参数错误。

## UC-04 发布帖子、评论与回复

- 参与者：注册用户
- 关联用户故事：US-09、US-10、US-11、US-13、US-14
- 前置条件：用户已登录，目标板块存在
- 后置结果：帖子或评论被保存并展示

基本流程：

1. 用户进入发布页，选择板块和帖子类型。
2. 用户填写标题、正文和股票代码标签。
3. 系统校验标题、正文、板块和登录状态。
4. 系统保存帖子并返回帖子详情。
5. 其他用户进入详情页发表评论。
6. 用户可针对某条评论进行楼中楼回复。

异常流程：

1. 未登录发布：返回 `401`。
2. 标题或正文为空：返回 `400`。
3. 父评论不属于当前帖子：返回业务错误。

## UC-05 点赞、收藏和搜索

- 参与者：注册用户、游客
- 关联用户故事：US-12、US-23、US-24
- 前置条件：目标帖子存在
- 后置结果：互动状态更新，搜索结果返回

基本流程：

1. 用户进入帖子详情页。
2. 登录用户点击点赞或收藏。
3. 系统使用 `(post_id, user_id)` 保证同一用户只保留一个状态。
4. 用户在搜索框输入关键词或股票代码。
5. 系统按标题、正文和股票代码返回帖子列表。
6. 用户输入片段时，系统返回搜索联想。

异常流程：

1. 未登录点赞收藏：返回 `401`。
2. 重复点击点赞或收藏：切换为取消状态。
3. 搜索无结果：前端展示空状态。

## UC-06 关注、粉丝和关注流

- 参与者：注册用户
- 关联用户故事：US-16、US-17、US-18
- 前置条件：用户已登录，目标用户存在
- 后置结果：关注关系建立或取消，关注流刷新

基本流程：

1. 用户进入其他用户主页。
2. 用户点击关注。
3. 系统校验不能关注自己。
4. 系统建立 `follower_id -> followee_id` 关系。
5. 用户进入关注列表或粉丝列表查看关系。
6. 用户打开关注流，系统只返回关注用户发布的帖子。

异常流程：

1. 关注自己：返回 `400`。
2. 目标用户不存在：返回 `404`。
3. 重复关注：保持幂等或提示已关注。

## UC-07 私信会话

- 参与者：注册用户
- 关联用户故事：US-19
- 前置条件：双方用户存在且已登录
- 后置结果：消息保存到双方会话，未读数更新

基本流程：

1. 用户 A 打开用户 B 的主页或消息页。
2. 用户 A 发送文本私信。
3. 系统保存消息并将 B 的未读数加一。
4. 用户 B 查看会话列表。
5. 用户 B 打开与 A 的会话，系统自动标记已读。
6. 用户 B 的未读数变为零。

异常流程：

1. 发送给自己：返回 `400`。
2. 内容为空：返回参数错误。
3. 目标用户不存在：返回 `404`。

## UC-08 管理员板块、帖子和用户管理

- 参与者：管理员
- 关联用户故事：US-25、US-26、US-27、US-28
- 前置条件：管理员已登录后台
- 后置结果：板块、帖子审核状态或用户状态更新

基本流程：

1. 管理员进入后台首页查看统计数据。
2. 管理员进入板块管理页新增、编辑或删除板块。
3. 管理员进入帖子管理页查看待审核或可疑内容。
4. 管理员对帖子执行通过、标记或驳回操作。
5. 管理员进入用户管理页，对用户执行警告、封禁、解封操作。
6. 系统记录处理结果并影响前台展示或用户权限。

异常流程：

1. 删除存在子板块或内容的板块：返回冲突错误。
2. 审核不存在的帖子：返回 `404`。
3. 管理动作缺少原因：根据动作类型要求补充原因。
