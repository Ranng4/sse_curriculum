# 后端接口设计

## 1. 接口规范

- 基础地址：`http://127.0.0.1:8000`
- 接口前缀：`/api/v1`
- 认证方式：`Authorization: Bearer <access_token>`
- 响应格式：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

## 2. 接口分组

### 2.1 Auth

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/v1/auth/register` | 用户注册 |
| POST | `/api/v1/auth/login` | 用户登录 |
| GET/POST | `/api/v1/auth/basic-verification` | 基础认证查询和提交 |
| GET/POST | `/api/v1/auth/real-name-verification` | 实名认证查询和提交 |
| GET/POST | `/api/v1/auth/professional-verification` | 专业认证查询和提交 |

## 3. 错误码约定

| HTTP状态 | 含义 |
| --- | --- |
| 400 | 参数错误或业务规则不满足 |
| 401 | 未登录、token 缺失或 token 失效 |
| 404 | 用户、板块、帖子、评论等资源不存在 |
| 409 | 资源冲突，例如重复注册、删除受限资源 |

## 4. 调用示例

注册后发布帖子：

```json
{
  "board_id": "board-id",
  "title": "新能源ETF周观察",
  "content": "关注成交量和资金净流入变化。",
  "post_type": "normal",
  "stock_codes": ["159915"]
}
```

搜索帖子：

```text
GET /api/v1/content/posts?keyword=ETF&sort=hot&limit=20
```
