# 安装部署文档

## 1. 环境要求

1. Python 3.13+
2. `uv` 包管理工具
3. Node.js 20+ 和 npm
4. Windows、Linux 或 macOS

## 2. 获取项目

```bash
git clone <your-repo-url>
cd sse_curriculum
```

## 3. 后端安装与启动

在项目根目录同步依赖：

```bash
uv sync
```

启动后端：

```bash
cd backend
uv run python -m app.main
```

默认服务地址：

```text
http://127.0.0.1:8000
```

健康检查：

```bash
curl http://127.0.0.1:8000/healthz
```

预期响应：

```json
{"status":"ok"}
```

## 4. 前端安装与启动

```bash
cd frontend
npm install
npm run dev
```

前端默认使用 Vite 开发服务器，启动后按终端输出地址访问。

## 5. 数据库说明

系统默认使用 SQLite。本地数据库文件默认位于：

```text
backend/data/soft_resign.sqlite3
```

如需指定 SQLite 路径，可设置环境变量：

```powershell
$env:SOFT_RESIGN_SQLITE_PATH="D:\data\soft_resign.sqlite3"
uv run python -m app.main
```

如需迁移到 MySQL，可参考：

```text
backend/sql/schema.sql
```

## 6. 运行测试

```powershell
cd backend
$env:UV_CACHE_DIR='D:\Codes\sse_curriculum\.uv-cache-report'
uv run python -m unittest discover -s tests -v
```

预期结果：

```text
Ran 10 tests
OK
```

## 7. 常见问题

1. `ModuleNotFoundError: No module named 'app'`：请确认从 `backend/` 目录运行测试或后端。
2. `uv` 缓存权限错误：设置一个新的项目内缓存目录，例如 `.uv-cache-report`。
3. `401` 鉴权失败：确认请求头格式为 `Authorization: Bearer <token>`。
4. 端口占用：修改 `backend/app/main.py` 中的端口或释放 `8000` 端口。
