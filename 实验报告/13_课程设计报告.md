# 股票基金投资论坛课程设计报告

## 1. 项目概述

本课程设计实现了一个股票基金投资论坛系统。系统面向投资交流场景，支持用户注册登录、个人资料、风险评估、论坛板块、帖子发布、评论互动、点赞收藏、热榜搜索、关注粉丝、私信沟通和后台管理。

项目采用前后端分离架构。前端使用 Vue 3 和 Vite，后端使用 Flask 3 和 Pydantic 2，数据存储使用 SQLite，并提供 MySQL 建表脚本。后端通过 RESTful API 对外提供能力，前端通过统一 API 封装调用接口。

## 2. 需求分析

根据指导书中“股票基金投资论坛”题目，系统需求被拆分为五类：

1. 用户系统：注册登录、认证、资料管理、投资偏好、风险评估。
2. 内容系统：板块分类、发帖、评论、点赞、收藏、长文、股票标签。
3. 社交系统：关注、粉丝、关注动态、私信。
4. 信息整合系统：首页信息流、热榜、搜索、搜索联想。
5. 管理运营系统：板块管理、内容审核、用户管理、数据统计。

需求阶段产出用户故事和交互场景，并按核心、重要、扩展划分优先级。课程设计阶段优先完成可运行闭环功能，将短信、OAuth、人脸识别、附件审核和群组作为扩展能力。

## 3. 系统设计

后端采用 API、Service、Repository、Model、Schema 分层结构。API 层负责路由和请求响应，Service 层负责业务规则，Repository 层负责数据访问，Model 层描述领域对象，Schema 层负责数据校验。

前端按页面和组件组织。页面覆盖首页、信息流、分区、板块、帖子详情、发帖、搜索、热榜、登录注册、个人中心、私信和后台管理。组件层提供帖子卡片、评论项、导航、加载、错误和空状态展示。

数据库围绕用户、认证、资料、风险评估、板块、帖子、评论、点赞、收藏、关注和私信等实体设计。点赞、收藏和关注关系使用联合主键思想，保证状态唯一。

## 4. 编码实现

已实现后端模块包括：

1. 认证和用户资料模块。
2. 适当性风险评估模块。
3. 论坛板块分区和 CRUD。
4. 帖子、评论、点赞、收藏、热榜、搜索联想和关注流。
5. 关注、粉丝、关注统计。
6. 私信、会话、未读数。
7. 后台统计、帖子审核、用户状态管理。

已实现前端页面包括首页、板块页、帖子详情页、发布页、搜索页、热榜页、登录注册页、个人中心、私信页和后台管理页。

关键业务规则包括：未登录用户不能互动，用户不能关注自己，用户不能给自己发私信，父评论必须属于同一帖子，重复点赞收藏需要保持唯一状态。

## 6. AI辅助过程

AI 被用于需求拆解、用户故事生成、交互场景草拟、架构设计建议、数据库脚本草案、接口文档模板、代码骨架、测试用例和报告初稿生成。

人工在关键节点进行了审查和修正：

1. 将 AI 初稿中的通用论坛需求调整为股票基金投资论坛需求。
2. 将架构从简单 MVC 调整为分层结构。
3. 为点赞、收藏、关注关系增加唯一约束思想。
4. 补充不能关注自己、不能私信自己、父评论归属校验等业务规则。
5. 修正测试环境、接口路径和报告中与实际代码不一致的表述。

## 7. 安装与运行

后端运行：

```bash
cd backend
uv run python -m app.main
```

前端运行：

```bash
cd frontend
npm install
npm run dev
```

健康检查：

```bash
curl http://127.0.0.1:8000/healthz
```

## 8. 项目总结

本项目完成了指导书要求的软件工程流程：需求分析、系统设计、编码实现、测试调试、部署说明和课程报告。系统具备可运行的前后端结构、较完整的业务模块和自动化测试验证。

不足之处在于真实第三方认证、短信邮件、人脸识别、附件审核、群组、高并发性能测试和前端组件测试尚未完整实现。后续可在现有分层架构基础上继续扩展。
