## 1. 项目概述

### 1.1 项目基本介绍

#### 📱 项目介绍

**股票基金投资论坛（SSE Investment Forum）** 是一个面向股票、基金投资者的专业交流社区平台。系统采用前后端分离的 Web 架构，前端基于 Vue 3 + Vite 构建现代化用户界面，后端基于 Flask 3 + Pydantic 2 提供 RESTful API 服务，数据存储支持 SQLite（本地运行）和 MySQL 8（生产部署）。

本论坛围绕投资交流的核心场景，提供以下核心能力：

1. **用户系统**：支持手机号、邮箱、第三方账号注册，基础认证 + 实名认证 + 专业认证三级认证体系，投资者适当性风险评估，个人资料与投资偏好管理。
2. **内容系统**：多板块分类讨论（A股、港股、美股、基金等），支持普通帖子、长文分析、实时讨论等多种发帖形式，点赞、收藏、多级评论（楼中楼）、@提及、股票代码标签。
3. **社交系统**：用户关注与粉丝机制、关注动态推送、私信沟通。
4. **信息整合**：首页信息流、热榜排行、全文搜索、搜索联想、个性化关注流。
5. **管理运营**：板块动态管理、内容审核（敏感词过滤）、用户行为管理（警告/禁言/封号）、论坛数据统计。

本系统严格遵循《软件工程理论与实践课程设计》指导书的五大实验模块要求，采用 AI 辅助（Human-In-the-Loop）模式完成需求分析、系统设计、编码实现、测试调试和文档撰写全过程。

#### 🚀 功能特性

1. **多级用户认证**：基础认证（手机/邮箱验证）→ 实名认证 → 专业认证（加V标识），满足不同层次用户需求。
2. **投资者适当性评估**：专业用户完成风险评估问卷，系统自动计算风险等级。
3. **多样化内容发布**：普通帖子、长文分析（富文本）、实时讨论，支持股票代码标签和图片。
4. **完整互动体系**：点赞、收藏、多级评论（楼中楼）、@提及功能。
5. **社交关系网络**：关注/粉丝机制、关注动态流、一对一私信。
6. **智能信息聚合**：首页信息流、热榜排行、全文搜索、股票代码搜索联想。
7. **后台管理面板**：论坛活跃度统计、帖子审核队列、用户状态管理。
8. **前后端分离**：Vue 3 前端 + Flask 3 后端 + RESTful API，架构清晰，易于扩展。
9. **双数据库支持**：本地 SQLite 开发 + MySQL 8 生产建表脚本。

## 2. 需求描述

### 2.1 功能性描述

#### 2.1.1 用户需求

##### UR-01 用户注册与认证（User Registration & Authentication）

**用户需求标识**：SSE-UR-AUTH

1. **用户需求**：游客通过手机号或邮箱完成注册，使用账号密码登录系统，成为论坛注册用户。
2. **分级认证**：
   - 基础认证：手机号/邮箱验证码验证
   - 实名认证：提交身份证信息进行身份核验
   - 专业认证：上传从业资格证、学历证明等，审核通过后获得加 V 标识

##### UR-02 个人资料管理（Profile Management）

**用户需求标识**：SSE-UR-PROFILE

1. **用户需求**：注册用户设置昵称、头像、个人简介、投资经验标签、关注市场领域。
2. **投资偏好**：配置关注的股票市场（A股/港股/美股/基金等）和风险偏好。
3. **隐私控制**：设置个人资料各字段的可见范围（公开/仅关注者/仅自己）。
4. **成就展示**：展示发帖数、精华帖数、影响力值和荣誉勋章。

##### UR-03 投资者适当性评估（Suitability Assessment）

**用户需求标识**：SSE-UR-SUITABILITY

1. **用户需求**：专业用户完成风险评估问卷，系统根据答案计算风险等级。
2. **问卷内容**：涵盖投资经验、风险承受能力、投资目标、财务状况等维度。
3. **结果输出**：保守型、稳健型、平衡型、成长型、进取型五种风险等级。

##### UR-04 论坛板块浏览（Forum Board Browsing）

**用户需求标识**：SSE-UR-BOARD

1. **用户需求**：用户浏览 A 股、港股、美股、基金等市场讨论区，以及价值投资、量化投资、基金投资等主题专区。
2. **分区导航**：市场讨论区、主题专区、公司研究专区、问答求助区四大分区。
3. **板块管理**：管理员可动态添加、删除和修改板块。

##### UR-05 发帖与内容发布（Post Publishing）

**用户需求标识**：SSE-UR-POST

1. **用户需求**：注册用户在指定板块发布帖子，分享投资观点或提出问题。
2. **帖子类型**：普通帖子（文字+图片）、长文分析（富文本）、实时讨论（简短文字）。
3. **股票标签**：关联股票代码，支持按股票检索相关帖子。
4. **附件支持**：上传 PDF、Excel 等分析报告。

##### UR-06 内容互动（Content Interaction）

**用户需求标识**：SSE-UR-INTERACT

1. **用户需求**：用户对帖子进行点赞、收藏、发表评论和回复评论。
2. **多级评论**：支持楼中楼评论结构，父评论归属校验。
3. **@提及**：在帖子和评论中 @ 其他用户，实现定向通知。
4. **转发**：将有价值的内容转发给其他用户。

##### UR-07 社交关系（Social Relations）

**用户需求标识**：SSE-UR-SOCIAL

1. **用户需求**：用户关注感兴趣的投资达人，查看关注动态和粉丝列表。
2. **特别关注**：星标重要用户，优先查看其内容。
3. **关注流**：按时间线查看关注用户发布的内容。
4. **私信沟通**：一对一发送文本私信，查看会话和未读消息。

##### UR-08 信息聚合与搜索（Information Aggregation）

**用户需求标识**：SSE-UR-SEARCH

1. **用户需求**：浏览首页信息流、热榜排行，按关键词或股票代码搜索帖子。
2. **信息流**：按时间倒序展示最新帖子，支持切换关注流。
3. **热榜**：基于互动指标（点赞、收藏、评论）的热门帖子排行。
4. **全文搜索**：支持关键词、股票代码搜索，按时间和热度排序。
5. **搜索联想**：输入股票代码或主题关键词时的自动补全。

##### UR-09 后台管理（Admin Panel）

**用户需求标识**：SSE-UR-ADMIN

1. **用户需求**：管理员审核帖子内容、管理用户状态、查看论坛活跃度统计。
2. **内容审核**：敏感词自动过滤 + 人工审核队列 + 举报处理。
3. **用户管理**：行为监控、警告、禁言、封号处理。
4. **数据统计**：论坛活跃度、热门话题、用户参与度分析。

#### 2.1.2 系统需求

##### SR-01 用户注册流程（SSE-SR-REGISTER）

1. **初始假设**：用户未注册，访问注册页面。
2. **正常状态**：
   - 用户输入手机号/邮箱和密码
   - 系统校验手机号/邮箱格式和唯一性
   - 创建用户记录并返回注册成功
   - 生成 access token，用户进入登录状态
3. **异常处理**：
   - 手机号/邮箱已注册 → 返回 409 Conflict
   - 密码不符合强度要求 → 返回 400 Validation Error
   - 注册方式不支持 → 返回 400 Validation Error

##### SR-02 用户登录流程（SSE-SR-LOGIN）

1. **初始假设**：用户已注册，访问登录页面。
2. **正常状态**：
   - 用户输入账号（手机号/邮箱）和密码
   - 系统验证凭据
   - 返回 access token 和用户基本信息
3. **异常处理**：
   - 账号不存在 → 返回 404 Not Found
   - 密码错误 → 返回 401 Unauthorized
   - Token 过期 → 返回 401 Unauthorized

##### SR-03 帖子发布流程（SSE-SR-CREATE-POST）

1. **初始假设**：用户已登录，进入发帖页面。
2. **正常状态**：
   - 用户选择目标板块
   - 填写标题、内容、帖子类型和股票代码
   - 系统进行敏感词检测
   - 创建帖子记录并返回帖子详情
3. **异常处理**：
   - 板块不存在或已停用 → 返回 400/404
   - 内容包含敏感词 → 返回 400 Validation Error
   - 未登录 → 返回 401 Unauthorized

##### SR-04 评论互动流程（SSE-SR-COMMENT）

1. **初始假设**：用户已登录，查看帖子详情。
2. **正常状态**：
   - 用户输入评论内容
   - 可选回复某条父评论（楼中楼）
   - 系统校验父评论归属
   - 创建评论记录
3. **异常处理**：
   - 帖子不存在 → 返回 404
   - 父评论不属于当前帖子 → 返回 400 Validation Error
   - 评论内容包含敏感词 → 返回 400 Validation Error

##### SR-05 点赞收藏流程（SSE-SR-LIKE-FAVORITE）

1. **初始假设**：用户已登录，浏览帖子。
2. **正常状态**：
   - POST 请求 = 点赞/收藏
   - DELETE 请求 = 取消点赞/收藏
   - 返回最新互动指标
3. **异常处理**：
   - 帖子不存在 → 返回 404
   - 未登录 → 返回 401
   - 重复点赞不报错，状态保持唯一

##### SR-06 关注流程（SSE-SR-FOLLOW）

1. **初始假设**：用户已登录，查看其他用户资料。
2. **正常状态**：
   - POST 请求 = 关注
   - DELETE 请求 = 取消关注
   - 不能关注自己
   - 返回关注状态和统计
3. **异常处理**：
   - 关注自己 → 返回 400 Validation Error
   - 目标用户不存在 → 返回 404

##### SR-07 私信流程（SSE-SR-MESSAGE）

1. **初始假设**：用户已登录，进入私信页面。
2. **正常状态**：
   - 发送消息给其他用户
   - 查看会话列表（按最新消息排序）
   - 查看与特定用户的会话详情
   - 查看未读消息数
3. **异常处理**：
   - 给自己发消息 → 返回 400 Validation Error
   - 接收方不存在 → 返回 404
   - 未登录 → 返回 401

##### SR-08 搜索流程（SSE-SR-SEARCH）

1. **初始假设**：用户访问搜索页面。
2. **正常状态**：
   - 输入关键词 → 匹配帖子标题和内容
   - 输入股票代码 → 匹配帖子的股票标签
   - 支持按最新/最热排序
   - 搜索联想返回匹配的股票代码、板块名和帖子标题
3. **异常处理**：
   - 无匹配结果 → 返回空列表

##### SR-09 后台审核流程（SSE-SR-ADMIN-REVIEW）

1. **初始假设**：管理员登录后台。
2. **正常状态**：
   - 查看帖子列表（支持按审核状态过滤）
   - 审查帖子内容
   - 标记为已审核通过或驳回
3. **异常处理**：
   - 非管理员用户 → 通过前端路由守卫限制

##### SR-10 用户管理流程（SSE-SR-ADMIN-USER）

1. **初始假设**：管理员登录后台。
2. **正常状态**：
   - 查看用户列表
   - 对违规用户执行警告/禁言/封号操作
   - 查看论坛统计数据
3. **异常处理**：
   - 操作不存在的用户 → 返回 404

### 2.2 非功能性需求

#### 2.2.1 性能需求

- **接口响应时间**：常规 CRUD 接口应在 200ms 内返回结果。
- **列表查询**：帖子列表、评论列表支持分页（默认 limit=20，最大 100）。
- **数据一致性**：点赞、收藏、关注关系使用唯一约束，保证状态一致性。
- **前端加载**：首屏加载时间应控制在 3 秒以内。

#### 2.2.2 安全性需求

- **认证鉴权**：所有受保护接口通过 Bearer Token 鉴权。
- **密码安全**：用户密码使用哈希存储，不明文保存。
- **输入校验**：所有用户输入通过 Pydantic Schema 校验。
- **内容审核**：敏感词过滤，防止违规内容发布。
- **XSS 防护**：前端输出进行适当转义处理。
- **SQL 注入防护**：使用参数化查询。

#### 2.2.3 可用性需求

- **错误提示**：接口返回统一 `{code, message, data}` 格式，便于前端处理。
- **状态展示**：前端覆盖加载中（LoadingSpinner）、空状态（EmptyState）、错误（ErrorMessage）三种 UI 状态。
- **登录持久化**：Token 存储于 localStorage，刷新页面保持登录状态。
- **路由守卫**：未登录用户自动跳转到登录页。

#### 2.2.4 可维护性需求

- **分层清晰**：后端 API-Service-Repository-Model-Schema 五层分离。
- **仓储抽象**：Repository 层屏蔽存储细节，支持 SQLite → MySQL 平滑切换。
- **统一响应**：所有接口使用统一的 `{code, message, data}` 响应结构。
- **文档完整**：覆盖需求、设计、接口、测试、安装、使用的全流程文档。

#### 2.2.5 可扩展性需求

- **模块化路由**：后端使用 Flask Blueprint 模块化注册路由。
- **服务容器**：依赖注入容器管理所有服务实例，便于单元测试和替换实现。
- **数据库兼容**：提供 SQLite（本地）和 MySQL 8（生产）两套建表方案。
- **认证体系可扩展**：预留微信、微博等第三方 OAuth 认证接口。

#### 2.2.6 用户体验需求

- **响应式设计**：前端适配桌面和移动端浏览。
- **即时反馈**：点赞、收藏、关注操作即时更新 UI 状态。
- **导航清晰**：顶部导航栏 + 分区层级结构，用户可快速定位目标板块。
- **搜索便捷**：支持搜索联想、股票代码快捷搜索。

## 3. 系统设计

### 3.1 架构设计

本系统采用前后端分离的分层架构模式，前端为单页应用（SPA），后端为 RESTful API 服务。

#### 3.1.1 表示层（Presentation Layer）

表示层是用户与系统交互的界面，由 Vue 3 单页应用实现，包括：

1. **页面视图（views/）**：按功能模块组织的页面组件。
   - 论坛页面：首页、信息流、热榜、分区、板块、搜索
   - 内容页面：发帖、帖子详情
   - 认证页面：登录、注册
   - 个人中心：我的资料、编辑资料、他人资料、风险评估
   - 私信页面：私信列表、会话详情
   - 后台管理：仪表盘、帖子审核、用户管理

2. **通用组件（components/）**：可复用的 UI 组件。
   - NavBar：顶部导航栏
   - PostCard：帖子列表卡片
   - CommentItem：评论展示项
   - LoadingSpinner：加载状态
   - ErrorMessage：错误提示
   - EmptyState：空数据状态

3. **路由管理（router/index.js）**：Vue Router 配置，包含路由守卫和懒加载。

4. **API 封装（utils/api.js）**：Axios 实例，统一处理请求拦截、Token 注入和错误处理。

#### 3.1.2 应用层（Application Layer）

应用层是系统的核心业务逻辑层，由 Flask 后端实现，包括：

1. **API 路由层（api/routers/）**：接收 HTTP 请求，参数校验，调用服务层，返回统一响应。
   - `auth.py`：注册、登录、认证状态
   - `profile.py`：资料查询和修改
   - `suitability.py`：风险问卷和评估
   - `forum.py`：板块分区和管理
   - `content.py`：帖子、评论、点赞、收藏、搜索、热榜
   - `social.py`：关注、粉丝、统计
   - `messages.py`：私信、会话、未读数
   - `admin.py`：后台统计、审核、用户管理

2. **业务服务层（services/）**：实现核心业务规则和流程控制。
   - 认证服务：密码验证、Token 管理
   - 内容服务：敏感词审核、父评论校验、互动指标计算
   - 社交服务：关注关系校验（不可自关注）
   - 私信服务：消息收发校验（不可自发自收）

3. **依赖注入（api/deps.py）**：ServiceContainer 管理所有服务实例和仓储实例。

#### 3.1.3 服务层（Service Layer）

服务层为业务层提供基础支撑：

1. **数据仓储层（repositories/）**：封装数据访问逻辑。
   - `sqlite_backend.py`：SQLite 底层操作（初始化、查询、写入）
   - 各业务仓储：User、Token、Forum、Content、Social、Message
   - 使用 JSON 序列化存储复杂字段（如 stock_codes、focus_markets）

2. **数据模型层（models/）**：定义核心领域对象。
   - `user.py`：User、UserAuth、UserProfile、PrivacySettings、SuitabilityAssessment 等
   - `content.py`：Post、Comment（使用 Python dataclass）
   - `forum.py`：ForumBoard

3. **数据校验层（schemas/）**：Pydantic 模型定义请求和响应结构。

4. **基础设施层（core/）**：异常类定义、枚举、安全工具。

### 3.2 界面原型设计

#### 3.2.1 首页（HomePage）

首页是论坛的入口页面，展示系统的主要功能入口和推荐内容。

- **顶部导航栏**：Logo、搜索入口、导航菜单（首页、信息流、热榜、分区）、用户菜单。
- **分区入口卡片**：A股市场、港股市场、美股市场、基金投资等快捷入口。
- **最新帖子列表**：按时间倒序展示近期帖子。
- **热榜侧栏**：展示当前热门帖子排行。

#### 3.2.2 分区页（SectionPage）

按分区类型展示下属板块。

- **分区标识**：市场讨论区、主题专区、公司研究、问答求助。
- **板块列表**：每个板块显示名称、简介、帖子数统计。
- **快捷发帖按钮**：进入板块后可直接发帖。

#### 3.2.3 板块页（BoardPage）

展示特定板块下的帖子列表。

- **板块信息**：板块名称、简介、帖子总数。
- **排序切换**：最新 / 最热。
- **帖子卡片列表**：标题、作者、摘要、互动数据（点赞/收藏/评论数）、时间。
- **分页加载**：支持滚动加载更多。

#### 3.2.4 帖子详情页（PostDetailPage）

展示帖子的完整内容和互动区域。

- **帖子正文**：标题、作者信息、发布时间、正文内容、股票代码标签。
- **互动按钮**：点赞（可切换）、收藏（可切换）、评论计数。
- **评论区**：评论列表（支持楼中楼结构），每条评论显示作者、内容、时间。
- **评论输入框**：支持回复特定评论。
- **相关帖子推荐**。

#### 3.2.5 发帖页（CreatePostPage）

用户发布新帖子的页面。

- **板块选择器**：下拉选择目标板块。
- **标题输入**：帖子标题输入框。
- **内容编辑器**：正文内容输入（支持基本格式）。
- **帖子类型选择**：普通 / 长文 / 实时。
- **股票代码标签**：添加关联的股票代码。
- **发布按钮**：提交前进行敏感词检测。

#### 3.2.6 搜索页（SearchPage）

全局搜索功能页面。

- **搜索框**：输入关键词或股票代码。
- **搜索联想**：下拉展示匹配建议。
- **搜索结果列表**：帖子卡片列表，支持按最新/最热排序。
- **筛选条件**：按市场、板块、时间范围过滤。

#### 3.2.7 热榜页（HotRankPage）

展示当前热门帖子排行。

- **时间范围切换**：今日 / 本周 / 全部。
- **排行列表**：按互动指标排序的帖子，展示排名、标题、互动数据。

#### 3.2.8 登录/注册页（LoginPage / RegisterPage）

用户认证页面。

- **登录表单**：账号（手机号/邮箱）、密码、登录按钮。
- **注册表单**：注册方式选择、账号、密码、确认密码。
- **跳转链接**：登录 ↔ 注册切换。

#### 3.2.9 个人中心（MyProfilePage / EditProfilePage）

用户个人信息管理页面。

- **资料展示**：头像、昵称、简介、投资经验标签、关注市场。
- **认证状态**：基础认证、实名认证、专业认证状态展示。
- **编辑入口**：修改基本资料、投资偏好、隐私设置。
- **风险评估入口**：完成/查看适当性评估问卷。

#### 3.2.10 私信页（MessageInbox / ConversationPage）

用户私信功能页面。

- **会话列表**：按最新消息排序，显示对方昵称、最新消息摘要、未读标记。

#### 3.3.3 系统流程分析

##### 用户注册与登录流程

```
用户 → 注册页面 → 输入注册信息 → POST /api/v1/auth/register
  → AuthService.register() → UserRepository.create()
  → 返回 access_token + user_info
  → 前端存储 token → 跳转首页

用户 → 登录页面 → 输入账号密码 → POST /api/v1/auth/login
  → AuthService.login() → 验证凭据 → 生成 Token
  → 返回 access_token + user_info
  → 前端存储 token → 跳转首页
```

##### 发帖与互动流程

```
用户 → 选择板块 → 填写标题/内容 → POST /api/v1/content/posts
  → ContentService.create_post()
    → content_filter.check() 敏感词检测
    → user_repository.get() 校验用户
    → forum_repository.get() 校验板块状态
    → content_repository.create_post() 持久化
  → 返回 PostView
  → 前端跳转到帖子详情页

其他用户 → 查看帖子 → GET /api/v1/content/posts/{id}
  → ContentService.get_post()
  → 返回 PostView（含 liked_by_me / favorited_by_me）

用户 → 点赞 → POST /api/v1/content/posts/{id}/like
  → ContentService.set_like(enabled=True)
  → content_repository.set_like() 联合主键保证唯一
  → 返回 PostMetricsView

用户 → 评论 → POST /api/v1/content/posts/{id}/comments
  → ContentService.create_comment()
    → 校验父评论归属（parent_comment_id → post_id 校验）
    → content_filter.check() 敏感词检测
  → 返回 CommentView
```

##### 关注与私信流程

```
用户A → 查看用户B资料 → POST /api/v1/social/follows/{B_id}
  → SocialService.follow()
    → _validate_follow() 不可自关注
    → social_repository.follow() 唯一约束
  → 返回 FollowActionResultView

用户A → 查看关注流 → GET /api/v1/content/feed?following_only=true
  → ContentService.list_feed(following_only=True)
    → social_repository.list_following() 获取关注列表
    → 过滤帖子（作者在关注列表中）
  → 返回过滤后的帖子列表

用户A → 给用户B发私信 → POST /api/v1/messages
  → MessageService.send_message()
    → 校验不可自发自收
    → message_repository.create() 持久化
  → 返回 MessageView
```

##### 搜索与热榜流程

```
用户 → 搜索 → GET /api/v1/content/posts?keyword=ETF&sort=hot
  → ContentService.list_posts()
    → 关键字匹配（title/content 不区分大小写）
    → 股票代码匹配（stock_codes 精确匹配）
    → sort=hot 按互动指标排序
    → sort=latest 按时间倒序
  → 返回 [PostView]

用户 → 搜索联想 → GET /api/v1/content/search/suggest?q=600
  → ContentService.search_suggestions()
    → 匹配股票代码、板块名、帖子标题
  → 返回 SearchSuggestView

用户 → 热榜 → GET /api/v1/content/hot-rank
  → ContentService.list_hot_rank()
    → content_repository.list_hot_posts()
    → engagement_score = like*2 + favorite*3 + comment
  → 返回 [PostView]
```

##### 后台管理流程

```
管理员 → 后台仪表盘 → GET /api/v1/admin/stats
  → AdminService.get_stats()
  → 返回 AdminStatsView（用户数、帖子数、评论数等）

管理员 → 帖子审核 → GET /api/v1/admin/posts?status=pending
  → AdminService.list_posts()
  → 审核操作 → PATCH /api/v1/admin/posts
  → AdminService.review_post(post_id, action)
  → 更新帖子审核状态

管理员 → 用户管理 → GET /api/v1/admin/users
  → 状态操作 → PATCH /api/v1/admin/users
  → AdminService.update_user_status(user_id, status)
  → 警告 / 禁言 / 封号 / 解封
```

```python
def get_current_user_id_from_request(request: Request) -> str:
    token = _extract_bearer_token(request)
    if token is None:
        raise UnauthorizedError("missing or invalid Authorization header")
    try:
```

```python
def create_post(self, user_id: str, request: PostCreateRequest) -> PostView:
    # 敏感词检测
    full_text = f"{request.title} {request.content}"
    check = content_filter.check(full_text)
    if not check.passed:
        raise ValidationError(content_filter.get_violation_message(check))

    self.user_repository.get(user_id)
    board = self.forum_repository.get(request.board_id)
    if not board.is_active:
        raise ValidationError("board is inactive")

    post = Post(
        author_id=user_id,
        board_id=request.board_id,
        title=request.title,
        content=request.content,
        post_type=request.post_type,
        stock_codes=request.stock_codes,
        image_urls=request.image_urls,
    )
    saved = self.content_repository.create_post(post)
    return self._to_post_view(saved, viewer_user_id=user_id)
```

**评论发布（含父评论归属校验）**：

```python
def create_comment(self, user_id: str, post_id: str,
                   request: CommentCreateRequest) -> CommentView:
    # 敏感词检测
    check = content_filter.check(request.content)
    if not check.passed:
        raise ValidationError(content_filter.get_violation_message(check))

    self.user_repository.get(user_id)
    self.content_repository.get_post(post_id)

    # 父评论归属校验：防止跨帖子回复
    if request.parent_comment_id:
        parent = self.content_repository.get_comment(request.parent_comment_id)
        if parent.post_id != post_id:
            raise ValidationError("parent comment does not belong to the post")

    comment = Comment(
        post_id=post_id,
        author_id=user_id,
        content=request.content,
        parent_comment_id=request.parent_comment_id,
    )
    saved = self.content_repository.create_comment(comment)
    return self._to_comment_view(saved)
```

**关注流过滤**：

```python
def list_feed(self, viewer_user_id=None, following_only=False, limit=20):
    posts = self.content_repository.list_posts()
    if following_only:
        if viewer_user_id is None:
            raise UnauthorizedError("login required for following feed")
        following = {item[0] for item in
                     self.social_repository.list_following(viewer_user_id)}
        following.add(viewer_user_id)  # 包含自己的帖子
        posts = [post for post in posts if post.author_id in following]
    posts.sort(key=lambda item: item.created_at, reverse=True)
    return [self._to_post_view(post, viewer_user_id) for post in posts[:limit]]
```

**互动指标计算**：

```python
def _metrics(self, post_id: str) -> PostMetricsView:
    like_count = self.content_repository.like_count(post_id)
    favorite_count = self.content_repository.favorite_count(post_id)
    comment_count = self.content_repository.comment_count(post_id)
    return PostMetricsView(
        like_count=like_count,
        favorite_count=favorite_count,
        comment_count=comment_count,
        engagement_score=like_count * 2 + favorite_count * 3 + comment_count,
    )
```

**搜索与过滤**：

```python
def list_posts(self, viewer_user_id=None, board_id=None, keyword=None,
               stock_code=None, sort="latest", limit=20):
    posts = self.content_repository.list_posts()

    if board_id:
        posts = [post for post in posts if post.board_id == board_id]
    if keyword:
        normalized = keyword.strip().lower()
        posts = [post for post in posts
                 if normalized in post.title.lower()
                 or normalized in post.content.lower()]
    if stock_code:
        normalized_code = stock_code.strip().upper()
        posts = [post for post in posts
                 if normalized_code in {item.upper() for item in post.stock_codes}]

    if sort == "hot":
        posts.sort(key=lambda item: (
            self.content_repository.engagement_score(item.id), item.created_at
        ), reverse=True)
    else:
        posts.sort(key=lambda item: item.created_at, reverse=True)

    return [self._to_post_view(post, viewer_user_id) for post in posts[:limit]]
```

### 6.2 经验与收获

1. **分层架构的价值**：API-Service-Repository 的三层分离使得业务规则、数据访问和接口处理各司其职，在测试和修改时体现出良好的隔离性。

2. **AI 辅助的边界**：AI 在初稿生成、代码骨架和结构化整理方面效率很高，但业务规则（如不可自关注、父评论归属校验、联合主键约束）和工程细节（如接口路径对齐、测试环境配置）必须由人工判断和修正。

3. **测试驱动信心**：10 项自动化测试覆盖核心业务闭环，每次修改后运行测试能快速验证功能完整性。

4. **文档先行**：在设计阶段完成接口文档和数据库设计，编码阶段有明确的目标和约束，减少了返工。

### 6.3 不足与改进方向

1. **第三方服务集成**：当前短信、邮件、OAuth、人脸识别为预留接口，后续可接入真实服务。
2. **实时通信**：私信和通知可升级为 WebSocket 实现实时推送。
3. **前端测试**：可引入 Vitest 或 Cypress 补充前端组件和端到端测试。
4. **性能优化**：可引入 Redis 缓存热榜和搜索联想，减少数据库查询。
5. **群组功能**：指导书中提到的投资主题群组功能尚未实现。
6. **附件审核**：PDF/Excel 附件的上传和审核流程可进一步完善。
7. **部署**：当前以本地运行为主，后续可部署到云服务器并配置 CI/CD。
