## 1. 项目概述

### 1.1 项目基本介绍

#### 项目介绍

**股票基金投资论坛（SSE Investment Forum）** 是一个面向股票、基金投资者的专业交流社区平台。系统采用前后端分离的 Web 架构，前端基于 Vue 3 + Vite 构建现代化用户界面，后端基于 Flask 3 + Pydantic 2 提供 RESTful API 服务，数据存储支持 SQLite（本地开发零配置运行）和 MySQL 8（生产部署，含完整建表脚本）。

本论坛围绕投资交流的核心场景，实现五大子系统：

```mermaid
graph LR
    A["股票基金投资论坛"]
    A --> B["用户系统"]
    A --> C["内容系统"]
    A --> D["社交系统"]
    A --> E["信息整合"]
    A --> F["管理运营"]
    B --> B1["注册登录"]
    B --> B2["三级认证"]
    B --> B3["资料管理"]
    B --> B4["风险评估"]
    C --> C1["板块分类"]
    C --> C2["发帖讨论"]
    C --> C3["点赞收藏"]
    C --> C4["楼中楼评论"]
    C --> C5["@提及"]
    D --> D1["关注粉丝"]
    D --> D2["关注流"]
    D --> D3["私信会话"]
    D --> D4["未读计数"]
    E --> E1["信息流"]
    E --> E2["热榜排行"]
    E --> E3["全文搜索"]
    E --> E4["搜索联想"]
    F --> F1["板块管理"]
    F --> F2["内容审核"]
    F --> F3["用户管理"]
    F --> F4["数据统计"]
```

1. **用户系统**：手机号/邮箱注册，基础认证 -> 实名认证 -> 专业认证三级体系，投资者适当性风险评估（C1-C5 五级），个人资料与投资偏好管理，隐私设置（PUBLIC/FOLLOWERS/PRIVATE）。

2. **内容系统**：四大分区多板块分类讨论（市场讨论/主题专区/公司研究/问答求助），普通帖子/长文分析/实时讨论三种发帖形式，点赞、收藏（POST/DELETE 幂等设计）、多级评论（parent_comment_id 楼中楼）、@mention 提及、股票代码标签。

3. **社交系统**：用户关注与粉丝机制（含不可自关注校验）、关注动态流（following_only feed）、一对一私信沟通（含不可自发自收校验 + 未读计数 + 自动已读）。

4. **信息整合系统**：首页信息流（全部/关注切换）、热榜排行（engagement_score = likes*2 + favorites*3 + comments）、全文搜索（keyword + stock_code）、搜索联想（动态词库 + 默认词库）。

5. **管理运营系统**：板块动态管理（CRUD + 层级校验）、内容审核（正则敏感词 8 大类别 + 白名单）、用户状态管理（警告/禁言/封号/解封）、论坛数据统计。

本系统严格遵循《软件工程理论与实践课程设计》指导书的五大实验模块，采用 AI 辅助（Human-In-the-Loop）模式完成全流程：AI 负责需求拆解、架构初稿、代码骨架、测试草案和文档初稿；人工负责需求边界确认、业务规则补充、接口对齐、测试环境配置和最终质量检查。

#### 功能特性

- 多级用户认证：基础认证（手机/邮箱验证）-> 实名认证 -> 专业认证（加V标识），满足不同层次用户需求。
- 投资者适当性评估：专业用户完成 3 题风险评估问卷（投资经验年限/亏损承受能力/投资目标），系统自动计算总分并映射 C1(保守型)到 C5(进取型)五级风险等级。
- 多样化内容发布：普通帖子（文字+图片+基础格式）、长文分析（富文本）、实时讨论（类似微博），支持股票代码标签和图片 URL。
- 完整互动体系：点赞/取消点赞（POST/DELETE 幂等设计）、收藏/取消收藏、多级评论（parent_comment_id 楼中楼）、@mention 提及功能。
- 社交关系网络：关注/取关（不可自关注校验）、粉丝列表、关注统计、关注动态流（following_only 过滤）、私信（不可自发自收校验+未读计数+自动已读）。
- 智能信息聚合：首页信息流（全部/关注切换）、热榜排行（engagement_score 排序）、全文搜索（keyword+stock_code）、搜索联想（动态+默认双词库）。
- 后台管理面板：论坛活跃度统计（用户/帖子/评论/今日活跃/待审核/Top10 板块）、帖子审核队列（通过/驳回/标记+原因）、用户状态管理（警告/禁言/封号/解封+原因）。
- 前后端分离架构：Vue 3 SPA（18 页面+6 通用组件）+ Flask 3 RESTful API（8 Blueprint/40+ 端点），API-Service-Repository-Model-Schema 五层分层。
- 双数据库支持：本地 SQLite（零配置，pickle BLOB+JSON 序列化）+ MySQL 8 完整建表脚本（15+ 表，FULLTEXT 索引，联合主键）。

#### 2.1.2 系统需求

以下按系统需求编号，使用 [初始假设 - 正常状态 - 异常处理 - 后置结果] 的格式详细描述各功能的系统级需求。

**SR-01 用户注册流程**

- 初始假设：用户未注册，访问注册页面。
- 正常状态：选择注册方式（phone/email）-> 输入账号/密码/昵称 -> 系统校验账号格式和唯一性、密码强度（>=6位）-> 创建 User + UserAuth + UserProfile + PrivacySettings -> 生成 access_token（secrets.token_urlsafe(32)，24h TTL）-> TokenRepository.save() 持久化 -> 返回 {access_token, user_id} -> 前端 localStorage 存储 -> 跳转首页。
- 异常处理：(1)手机号/邮箱已注册 -> 409 Conflict；(2)密码不足6位 -> 400；(3)注册方式不合法 -> 400；(4)昵称为空 -> 400。

**SR-02 用户登录流程**

- 初始假设：用户已注册，访问登录页面。
- 正常状态：输入账号+密码 -> AuthService.login() 按 phone/email/open_id 查找 -> security.verify_password() 验证 PBKDF2-HMAC-SHA256（240K迭代）-> 生成新 access_token -> 返回 token+用户信息。
- 异常处理：(1)账号不存在 -> 404；(2)密码错误 -> 401；(3)Token过期（>24h）-> 401，前端自动清除token跳转登录页。

**SR-03 帖子发布流程**

- 初始假设：用户已登录，目标板块存在且活跃（is_active=true）。
- 正常状态：选择板块 -> 选择帖子类型（normal/longform/realtime）-> 填写标题/正文/股票代码 -> ContentFilter.check() 敏感词检测 -> 校验 board.is_active -> 创建 Post（UUID主键，UTC时间戳）-> 持久化 -> 返回 PostView（含作者/板块名/互动指标）。
- 异常处理：(1)内容含敏感词（股票操纵/内幕交易/赌博/色情/假证等）-> 400+违规原因；(2)板块停用 -> 400；(3)标题空/超200字符 -> 400；(4)未登录 -> 401。

**SR-04 评论互动流程**

- 初始假设：用户已登录，目标帖子存在。
- 正常状态：输入评论内容 -> 可选 parent_comment_id（楼中楼）-> ContentFilter.check() 敏感词检测 -> 若有 parent_comment_id：校验父评论.post_id == 当前post_id（防止跨帖回复）-> 创建 Comment -> 返回 CommentView。
- 异常处理：(1)父评论不属于当前帖子 -> 400 "parent comment does not belong to the post"；(2)含敏感词 -> 400；(3)帖子不存在 -> 404；(4)未登录 -> 401。

**SR-05 点赞收藏流程**

- 初始假设：用户已登录，目标帖子存在。
- 正常状态：POST = 点赞/收藏（INSERT OR IGNORE 到联合主键表），DELETE = 取消（DELETE FROM）。返回 PostMetricsView（like_count, favorite_count, comment_count, engagement_score = like*2+fav*3+comment）。幂等设计，重复操作不报错。
- 异常处理：(1)帖子不存在 -> 404；(2)未登录 -> 401。

**SR-06 关注流程**

- 初始假设：用户已登录，目标用户存在。
- 正常状态：POST /social/follows/{target} = 关注，DELETE = 取关。\_validate_follow() 确保 follower_id != followee_id。返回 FollowActionResultView（is_following + following_count + follower_count_of_target）。
- 异常处理：(1)关注自己 -> 400 "cannot follow yourself"；(2)目标用户不存在 -> 404。

**SR-07 私信流程**

- 初始假设：双方用户存在且已登录。
- 正常状态：发送方 POST /messages -> send() 校验 from_id != to_id -> 创建 Message -> 接收方未读数+1。接收方 GET /messages/conversations -> 会话列表（按最新消息排序，含对方昵称+最后消息+未读数）。GET /messages/conversations/{partner} -> 会话详情 -> 自动标记已读 -> 未读数清零。
- 异常处理：(1)自发自收 -> 400；(2)内容为空 -> 400；(3)接收方不存在 -> 404。

**SR-08 搜索与热榜流程**

- 初始假设：用户访问搜索页或热榜页。
- 正常状态：搜索：GET /content/posts?keyword=ETF&sort=hot -> 关键字不区分大小写匹配 title/content -> 股票代码精确匹配 stock_codes 集合 -> sort=latest 按时间倒序，sort=hot 按 engagement_score 降序。搜索联想：GET /content/search/suggest?q=600 -> 动态匹配（所有帖子stock_codes+titles+板块names）+ 默认词库（9个关键词）-> 前缀匹配 -> Top10。热榜：GET /content/hot-rank -> engagement_score 排序。
- 异常处理：(1)无结果 -> 返回 []；(2)sort参数非法 -> 400。

**SR-09 后台审核流程**

- 初始假设：管理员登录后台。
- 正常状态：GET /admin/stats -> 统计（用户/帖子/评论总数/今日活跃/待审核/被标记/Top10板块）。GET /admin/posts?status=pending -> 待审核列表。PATCH /admin/posts/{id} -> approve/reject/flag + reason。
- 异常处理：(1)帖子不存在 -> 404；(2)操作类型非法 -> 400。

**SR-10 用户管理流程**

- 初始假设：管理员登录后台。
- 正常状态：GET /admin/users（支持 keyword 搜索）-> PATCH /admin/users/{id} -> warn（警告，记录但不禁言）/mute（禁言，禁止发帖评论）/ban（封号，完全禁用）/unban（解封）+ reason。
- 异常处理：(1)用户不存在 -> 404；(2)操作类型非法 -> 400。

## 3. 系统设计

### 3.1 架构设计

本系统采用前后端分离的分层架构模式。前端为 Vue 3 单页应用（SPA），后端为 Flask 3 RESTful API 服务。

#### 3.1.1 表示层（Presentation Layer）

表示层是用户与系统交互的界面，由 Vue 3 单页应用实现：

1. **页面视图（views/）**：18 个页面组件，覆盖论坛（6页：首页/信息流/热榜/分区/板块/搜索）、内容（2页：发帖/帖子详情）、认证（2页：登录/注册）、个人中心（4页：我的资料/编辑/他人资料/风险评估）、私信（2页：会话列表/会话详情）和后台管理（3页：仪表盘/帖子审核/用户管理）六大模块。
2. **通用组件（components/）**：6 个可复用组件。NavBar（顶部导航，根据登录状态显示不同菜单，支持移动端汉堡菜单）、PostCard（帖子卡片，标题/摘要/作者/互动数据/股票标签）、CommentItem（评论项，支持楼中楼缩进嵌套）、LoadingSpinner（旋转加载动画）、ErrorMessage（错误图标+消息+重试按钮）、EmptyState（空图标+提示文字）。
3. **路由管理（router/index.js）**：Vue Router 4，19 条路由，全部使用懒加载。beforeEach 路由守卫检查 localStorage 中的 token，未登录用户仅可访问 login/register。
4. **状态管理（store/auth.js）**：Pinia 管理认证状态（token/userId/isLoggedIn），localStorage 持久化。
5. **API 封装（utils/api.js）**：Axios 实例，baseURL=/api/v1，请求拦截器注入 Bearer Token，响应拦截器自动解包 data + 401 自动清理 token。

#### 3.1.2 应用层（Application Layer）

应用层是系统的核心业务逻辑层，由 Flask 3 后端实现：

1. **API 路由层（api/routers/）**：8 个 Blueprint 模块，40+ 个 RESTful 端点。每个路由处理函数负责参数解析（request.get_json / request.args）、Pydantic 校验、调用服务层、返回统一 `{code, message, data}` 响应。
2. **业务服务层（services/）**：11 个服务类。AuthService（注册分派+登录验证+Token管理）、ContentService（发帖/评论/互动/搜索/热榜，含敏感词检测+父评论归属校验）、SocialService（关注/取关，含不可自关注校验）、MessageService（私信/会话/未读，含不可自发自收校验）、AdminService（统计/审核/用户管理）、ForumService（板块CRUD/分区聚合+层级校验）等。
3. **依赖注入（api/deps.py）**：ServiceContainer 单例，create*service_container() 工厂函数按依赖顺序（仓储->服务）创建并注入。提供 get*\*\_service() 获取函数和 Token 鉴权提取函数（get_current_user_id_from_request 强制 / get_optional_current_user_id_from_request 可选）。

#### 3.1.3 服务层（Service Layer）

服务层为业务层提供基础支撑：

1. **数据仓储层（repositories/）**：7 个仓储类 + SQLite 后端。sqlite_backend.py 提供线程安全连接管理（threading.local）+ 自动建表 + pickle BLOB 序列化。各业务仓储封装 CRUD 操作，使用 JSON 序列化存储复杂字段（stock_codes/focus_markets 等列表）。
2. **数据模型层（models/）**：Python dataclass 领域对象。User（含 UserAuth/UserProfile/PrivacySettings/SuitabilityAssessment/RealNameVerification/ProfessionalVerification）、Post（PostType 枚举：NORMAL/LONGFORM/REALTIME）、Comment（parent_comment_id 支持楼中楼）、ForumBoard（category/market/parent_id 层级）。UUID 主键 + UTC 时间戳。
3. **数据校验层（schemas/）**：Pydantic v2 模型，9 个模块。Request Schema 定义输入约束（Field min_length/max_length + AfterValidator），Response View 定义输出结构。
4. **基础设施层（core/）**：errors.py 五级异常体系（UserSystemError -> ValidationError/NotFoundError/ConflictError/UnauthorizedError），error_handlers.py 统一映射 HTTP 状态码。enums.py 全部枚举。security.py 密码哈希 + Token 生成。

### 3.2 界面原型设计

#### 3.2.1 首页（HomePage）

路由: `/`

论坛入口页面。顶部导航栏（Logo + 首页/动态/热榜导航 + 搜索入口 + 用户菜单）。中部：四大分区入口卡片（市场讨论区/主题专区/公司研究专区/问答求助区，每张卡片含简介）。下部：最新帖子列表（PostCard 组件，按发布时间倒序）。右侧：热榜侧栏（Top 8 热门帖子，含 engagement_score）。

#### 3.2.2 信息流页（FeedPage）

路由: `/feed`

最新帖子信息流。顶部标签切换（全部/关注），右上角发帖按钮（仅登录用户可见）。帖子列表使用 PostCard 组件，支持滚动分页。关注模式调用 following_only=true 参数，过滤仅显示关注用户和自己的帖子。

#### 3.2.3 分区页（SectionPage）

路由: `/sections/:category`

展示指定分区（market/topic/company_research/qa）下属的板块列表。每个板块显示：名称、简介描述、帖子总数统计。点击板块卡片进入板块页（BoardPage）。

#### 3.2.4 板块页（BoardPage）

路由: `/boards/:boardId`

展示指定板块下的帖子列表。顶部板块信息（名称 + 描述 + 帖子总数）。排序标签（最新/最热）。PostCard 列表。右上角发帖按钮。支持返回上一级。

#### 3.2.5 帖子详情页（PostDetailPage）

路由: `/posts/:postId`

展示帖子完整内容。正文区：板块徽章 + 标题 + 作者（可点击跳转主页）+ 发布时间 + 正文 + 股票代码标签（蓝色标签，如 600519）。互动栏：点赞按钮（心形图标，可切换，显示计数）+ 收藏按钮（星形图标，可切换，显示计数）+ 评论计数。评论区：CommentItem 嵌套展示楼中楼（缩进层级），按时间排序。底部：评论输入框（登录用户可见，支持回复特定评论 + @mention）。未登录用户显示"登录后参与互动"提示。

#### 3.2.6 发帖页（CreatePostPage）

路由: `/posts/new`

发帖表单。板块选择下拉框（列出所有活跃板块）。帖子类型选择按钮组（普通/长文/实时，带说明）。标题输入框。正文输入区（多行文本，含字符计数显示）。股票代码标签输入（逗号分隔，如 600519,000001）。图片 URL 输入（可选）。发布按钮（提交前敏感词检测，失败时弹窗显示具体违规原因和违规词汇）。

#### 3.2.7 搜索页（SearchPage）

路由: `/search`

全局搜索功能。顶部搜索输入框（输入时触发搜索联想下拉列表，显示匹配的股票代码/板块名/帖子标题）。排序切换（最新/最热）。搜索结果列表（PostCard）。搜索结果为空时显示 EmptyState 组件。

#### 3.2.8 热榜页（HotRankPage）

路由: `/hot`

热门帖子排行页面。排行列表：排名序号（1-3 名用金色/银色/铜色高亮）+ 帖子信息（PostCard）+ engagement_score 展示。基于 like_count*2 + favorite_count*3 + comment_count 排序。最多展示 50 条。

#### 3.2.9 登录/注册页

路由: `/auth/login`, `/auth/register`

AuthLayout 居中卡片布局（无导航栏）。登录页：应用标题 + 账号输入框 + 密码输入框 + 登录按钮 + 没有账号？去注册链接。注册页：注册方式切换标签（手机号/邮箱）+ 账号输入框 + 密码输入框 + 昵称输入框 + 验证码输入框（当前使用固定验证码 1234）+ 注册按钮 + 已有账号？去登录链接。

#### 3.2.10 个人中心（MyProfilePage 等）

路由: `/profile/*`

我的资料页：头像（用户名首字母圆形图标）+ 昵称 + 简介 + 统计栏（关注数/粉丝数/发帖数/影响力值）+ 成就徽章 + 标签页（基本信息/投资偏好/隐私设置/认证状态）。编辑资料页：基本信息表单（昵称/头像URL/简介/经验标签）+ 投资偏好表单（关注市场多选复选框：A股/港股/美股/基金/债券/期货 + 风险偏好下拉选择）。他人资料页（遵守隐私设置的公开资料 + 关注/取关按钮）。

#### 3.2.11 风险评估页（SuitabilityPage）

路由: `/suitability`

投资者适当性评估。已完成评估时显示：风险等级（C1保守型/C2稳健型/C3平衡型/C4成长型/C5进取型）+ 得分 + 评估时间。未完成评估时显示：3 道题目（投资经验年限/亏损承受上限/投资目标），每题 4 个选项，选择后提交，实时计算总分并显示对应风险等级。

#### 3.2.12 私信页

路由: `/messages/*`

会话列表：对方头像（首字母）+ 昵称 + 最新消息摘要（单行截断）+ 时间戳 + 未读数量徽标（红色圆点+数字），按最新消息时间倒序。会话详情：消息气泡列表（左侧灰色=对方发送，右侧蓝色=自己发送）+ 每条消息的时间戳 + 页面加载时自动滚动到底部 + 自动标记已读。底部：消息输入框 + 发送按钮。

#### 3.2.13 后台管理（AdminDashboard 等）

路由: `/admin/*`

仪表盘：4 个统计卡片（用户总数/帖子总数/评论总数/今日活跃度，带图标和数值）+ 待审核数和被标记数卡片（橙红色警示）+ Top10 活跃板块排行榜（板块名+帖子数，横向条形图）。帖子审核页：状态过滤标签行（全部/待审核/已标记/已通过/已驳回）+ 帖子列表表格（作者/标题/板块/发布时间/状态）+ 操作按钮（通过绿/驳回红/标记橙）+ 原因输入弹窗。用户管理页：搜索框（按昵称/手机号搜索）+ 用户列表表格（昵称/手机号/注册时间/状态）+ 操作按钮（警告黄/禁言橙/封号红/解封绿）+ 原因输入弹窗。

**错误码约定**

| HTTP 状态码 | 含义         | 触发场景                                                                     |
| ----------- | ------------ | ---------------------------------------------------------------------------- |
| 200         | 成功         | 正常查询、更新、删除操作                                                     |
| 201         | 创建成功     | 注册、发帖、评论、点赞、收藏、关注                                           |
| 400         | 请求参数错误 | Pydantic 校验失败 / 业务规则违反（自关注/自发自收/敏感词/跨帖回复/板块停用） |
| 401         | 未授权       | 缺少 Authorization 头 / Token 无效或过期（24h）                              |
| 404         | 资源不存在   | 用户/板块/帖子/评论/会话 不存在                                              |
| 409         | 资源冲突     | 重复注册（手机号/邮箱已存在）                                                |

#### 4.1.10 backend/app/services/auth_service.py - 认证服务完整实现

认证服务是用户系统的核心，负责处理手机/邮箱/第三方注册、账号登录、三级认证（基础/实名/专业）等全部认证流程。

```python
from __future__ import annotations

import hashlib
from datetime import UTC, datetime

from app.core.enums import ProfessionalDocType, RegisterMethod, VerificationStatus
from app.core.errors import ConflictError, ValidationError
from app.core.security import (
    generate_access_token, hash_password, mask_id_number,
    token_expire_time, verify_password,
)
from app.models.user import ProfessionalDocument, User, UserAuth, UserProfile
from app.repositories.token_repository import AccessTokenRecord, InMemoryTokenRepository
from app.repositories.user_repository import InMemoryUserRepository
from app.schemas.auth import (
    AuthTokenData, LoginRequest, ProfessionalVerificationView,
    RealNameVerificationView, RegisterByEmailRequest, RegisterByPhoneRequest,
    RegisterByThirdPartyRequest, RegisterRequest,
    SubmitProfessionalVerificationRequest, SubmitRealNameVerificationRequest,
    UserAuthView, VerifyBasicRequest,
)


class AuthService:
    def __init__(
        self,
        user_repository: InMemoryUserRepository,
        token_repository: InMemoryTokenRepository,
    ) -> None:
        self.user_repository = user_repository
        self.token_repository = token_repository

    def register(self, request: RegisterRequest) -> tuple[User, AuthTokenData]:
        """注册分派：根据 method 字段分发到 phone/email/third_party"""
        if isinstance(request, RegisterByPhoneRequest):
            user = self._register_by_phone(request)
        elif isinstance(request, RegisterByEmailRequest):
            user = self._register_by_email(request)
        else:
            user = self._register_by_third_party(request)
        token = self._issue_token(user.id)
        return user, token

    def _register_by_phone(self, request: RegisterByPhoneRequest) -> User:
        self._validate_verification_code(request.verification_code)
        if self.user_repository.find_by_phone(request.phone):
            raise ConflictError("phone already registered")
        user = User(
            register_method=RegisterMethod.PHONE,
            auth=UserAuth(
                phone=request.phone, phone_verified=True,
                password_hash=hash_password(request.password),
            ),
            profile=UserProfile(nickname=request.nickname),
        )
        return self.user_repository.create(user)

    def _register_by_email(self, request: RegisterByEmailRequest) -> User:
        self._validate_verification_code(request.verification_code)
        if self.user_repository.find_by_email(request.email):
            raise ConflictError("email already registered")
        user = User(
            register_method=RegisterMethod.EMAIL,
            auth=UserAuth(
                email=request.email, email_verified=True,
                password_hash=hash_password(request.password),
            ),
            profile=UserProfile(nickname=request.nickname),
        )
        return self.user_repository.create(user)

    def _register_by_third_party(self, request: RegisterByThirdPartyRequest) -> User:
        open_id = self._resolve_open_id(request.method, request.access_token)
        if request.method == RegisterMethod.WECHAT and \
           self.user_repository.find_by_wechat_open_id(open_id):
            raise ConflictError("wechat account already registered")
        if request.method == RegisterMethod.WEIBO and \
           self.user_repository.find_by_weibo_open_id(open_id):
            raise ConflictError("weibo account already registered")
        auth = UserAuth()
        if request.method == RegisterMethod.WECHAT:
            auth.wechat_open_id = open_id
        if request.method == RegisterMethod.WEIBO:
            auth.weibo_open_id = open_id
        user = User(register_method=request.method, auth=auth,
                    profile=UserProfile(nickname=request.nickname))
        return self.user_repository.create(user)

    def login(self, request: LoginRequest) -> AuthTokenData:
        """登录：按 phone/email/wechat/weibo 顺序查找用户"""
        user = (
            self.user_repository.find_by_phone(request.account)
            or self.user_repository.find_by_email(request.account)
            or self.user_repository.find_by_wechat_open_id(request.account)
            or self.user_repository.find_by_weibo_open_id(request.account)
        )
        if user is None:
            raise ValidationError("account or password incorrect")
        if not user.auth.password_hash:
            raise ValidationError("password login not supported for this account")
        if not verify_password(request.password, user.auth.password_hash):
            raise ValidationError("account or password incorrect")
        return self._issue_token(user.id)

    def verify_basic(self, user_id: str, request: VerifyBasicRequest) -> UserAuthView:
        """基础认证：手机验证码/邮箱验证码"""
        user = self.user_repository.get(user_id)
        code_given = bool(request.phone_code) or bool(request.email_code)
        if not code_given:
            raise ValidationError("phone_code or email_code is required")
        if request.phone_code and user.auth.phone:
            self._validate_verification_code(request.phone_code)
            user.auth.phone_verified = True
        if request.email_code and user.auth.email:
            self._validate_verification_code(request.email_code)
            user.auth.email_verified = True
        user.touch()
        saved = self.user_repository.save(user)
        return self.to_auth_view(saved)

    def submit_real_name_verification(
        self, user_id: str, request: SubmitRealNameVerificationRequest,
    ) -> RealNameVerificationView:
        """实名认证：提交姓名+身份证号+人脸（预留KYC接口）"""
        user = self.user_repository.get(user_id)
        record = user.real_name_verification
        record.status = VerificationStatus.PENDING
        record.legal_name = request.legal_name
        record.id_number = request.id_number
        record.submitted_at = datetime.now(UTC)
        record.reviewed_at = None
        record.rejection_reason = None
        # TODO: call OCR + real-name KYC provider
        if request.enable_face_recognition:
            record.face_verified = self._face_recognition_stub(
                request.face_image_url, request.legal_name, request.id_number)
        else:
            record.face_verified = False
        user.touch()
        saved = self.user_repository.save(user)
        return self.to_real_name_view(saved)

    def submit_professional_verification(
        self, user_id: str, request: SubmitProfessionalVerificationRequest,
    ) -> ProfessionalVerificationView:
        """专业认证：上传从业资质文件（预留审核队列接口）"""
        user = self.user_repository.get(user_id)
        verification = user.professional_verification
        verification.documents = [
            ProfessionalDocument(
                doc_type=self._to_doc_type(doc.doc_type),
                file_name=doc.file_name, file_url=doc.file_url,
            ) for doc in request.documents
        ]
        verification.status = VerificationStatus.PENDING
        verification.reviewed_at = None
        verification.rejection_reason = None
        verification.verified_by = None
        # TODO: call anti-fraud + manual audit queue service
        user.touch()
        saved = self.user_repository.save(user)
        return self.to_professional_view(saved)

    def get_user_by_token(self, token: str) -> User:
        """Token鉴权：验证有效性 -> 返回User或抛异常"""
        record = self.token_repository.get(token)
        if record.expired():
            self.token_repository.revoke(token)
            raise ValidationError("token expired")
        return self.user_repository.get(record.user_id)

    def _issue_token(self, user_id: str) -> AuthTokenData:
        """签发Token：secrets.token_urlsafe(32)，24h有效期"""
        token = generate_access_token()
        expires_at = token_expire_time()
        self.token_repository.save(
            AccessTokenRecord(token=token, user_id=user_id, expires_at=expires_at))
        return AuthTokenData(access_token=token, expires_at=expires_at)

    def _validate_verification_code(self, code: str) -> None:
        if len(code.strip()) < 4:
            raise ValidationError("verification code is invalid")
        # TODO: call sms/email verification provider

    def _resolve_open_id(self, method: RegisterMethod, access_token: str) -> str:
        """第三方OAuth open_id解析（SHA-256 stub，预留微信/微博API）"""
        if len(access_token.strip()) < 8:
            raise ValidationError("invalid third-party access token")
        # TODO: use official WeChat/Weibo OAuth API to exchange open_id
        prefix = "wx" if method == RegisterMethod.WECHAT else "wb"
        digest = hashlib.sha256(access_token.encode("utf-8")).hexdigest()[:24]
        return f"{prefix}_{digest}"

    def _face_recognition_stub(self, face_image_url, legal_name, id_number) -> bool:
        if not face_image_url:
            return False
        # TODO: integrate with face liveness and identity compare service
        return True

    def _to_doc_type(self, doc_type: str) -> ProfessionalDocType:
        mapping = {d.value: d for d in ProfessionalDocType}
        return mapping.get(doc_type, ProfessionalDocType.OTHER)

    @staticmethod
    def to_auth_view(user: User) -> UserAuthView:
        return UserAuthView(
            phone=user.auth.phone, phone_verified=user.auth.phone_verified,
            email=user.auth.email, email_verified=user.auth.email_verified,
            basic_verified=user.auth.basic_verified(),
            register_method=user.register_method)

    @staticmethod
    def to_real_name_view(user: User) -> RealNameVerificationView:
        v = user.real_name_verification
        return RealNameVerificationView(
            status=v.status, legal_name=v.legal_name,
            id_number_masked=mask_id_number(v.id_number),
            face_verified=v.face_verified, submitted_at=v.submitted_at,
            reviewed_at=v.reviewed_at, rejection_reason=v.rejection_reason)

    @staticmethod
    def to_professional_view(user: User) -> ProfessionalVerificationView:
        v = user.professional_verification
        return ProfessionalVerificationView(
            status=v.status, has_v_badge=v.has_v_badge,
            documents=[{
                "doc_type": d.doc_type.value,
                "file_name": d.file_name, "file_url": d.file_url,
                "uploaded_at": d.uploaded_at,
            } for d in v.documents],
            reviewed_at=v.reviewed_at, rejection_reason=v.rejection_reason)
```

**关键设计要点：**
- **注册分派模式**：使用 Pydantic 的 discriminated union（`RegisterRequest`），根据 `method` 字段自动路由到 phone/email/third_party 三种处理函数
- **冲突检测**：注册前查询 phone/email 唯一性，已存在则抛 `ConflictError`（HTTP 409）
- **登录多通道**：`login()` 按 phone → email → wechat → weibo 顺序尝试匹配账号
- **密码安全**：PBKDF2-HMAC-SHA256 哈希存储（240,000 次迭代），使用 `secrets.token_urlsafe(32)` 生成 Token
- **Token 生命周期**：24 小时 TTL，过期自动吊销，前端 401 拦截自动清理
- **第三方预留**：微信/微博 OAuth 使用 SHA-256 stub 生成 open_id，人脸识别 stub 始终返回 True，均预留 TODO 标记供后续接入

#### 4.1.11 backend/app/services/message_service.py - 私信服务完整实现

私信服务处理消息收发、会话聚合和未读计数，核心约束：不可给自己发私信。

```python
"""Message service — send, receive, conversation queries."""

from __future__ import annotations

from app.core.errors import NotFoundError, ValidationError
from app.repositories.message_repository import Message, MessageRepository
from app.repositories.user_repository import InMemoryUserRepository
from app.schemas.message import (
    ConversationPreview, MessageView, SendMessageRequest, UnreadCountView,
)


class MessageService:
    def __init__(
        self, message_repository: MessageRepository,
        user_repository: InMemoryUserRepository,
    ) -> None:
        self.msg_repo = message_repository
        self.user_repo = user_repository

    def send(self, from_id: str, req: SendMessageRequest) -> MessageView:
        """发送私信，核心校验：from_id != to_id（不可自发自收）"""
        if from_id == req.to_user_id:
            raise ValidationError("cannot send message to yourself")
        self.user_repo.get(from_id)
        self.user_repo.get(req.to_user_id)
        msg = Message(
            from_id=from_id, to_id=req.to_user_id,
            content=req.content, image_url=req.image_url, is_read=False)
        saved = self.msg_repo.create(msg)
        return self._to_view(saved)

    def get_conversation(
        self, user_id: str, partner_id: str, limit: int = 50
    ) -> list[MessageView]:
        """获取与指定用户的会话详情，自动标记已读"""
        self.user_repo.get(user_id)
        self.user_repo.get(partner_id)
        self.msg_repo.mark_read(user_id, partner_id)  # 自动已读，未读数清零
        msgs = self.msg_repo.list_conversation(user_id, partner_id, limit=limit)
        return [self._to_view(m) for m in msgs]

    def list_conversations(self, user_id: str) -> list[ConversationPreview]:
        """会话列表：按最新消息排序，含对方昵称+最后消息+未读数"""
        self.user_repo.get(user_id)
        convos = self.msg_repo.list_conversations(user_id)
        results = []
        for c in convos:
            try:
                partner = self.user_repo.get(c["partner_id"])
                nickname = partner.profile.nickname
            except NotFoundError:
                nickname = "unknown"  # 冗余容错
            results.append(ConversationPreview(
                partner_id=c["partner_id"], partner_nickname=nickname,
                unread_count=c["unread_count"], last_message=c["last_message"],
                last_time=c["last_time"], last_from_me=c["last_from_me"]))
        return results

    def get_unread_count(self, user_id: str) -> UnreadCountView:
        count = self.msg_repo.unread_count(user_id)
        return UnreadCountView(unread_count=count)

    def _to_view(self, msg: Message) -> MessageView:
        try:
            from_user = self.user_repo.get(msg.from_id)
            nick = from_user.profile.nickname
        except NotFoundError:
            nick = "unknown"
        return MessageView(
            id=msg.id, from_id=msg.from_id, from_nickname=nick,
            to_id=msg.to_id, content=msg.content, image_url=msg.image_url,
            created_at=msg.created_at, is_read=msg.is_read)
```

**私信系统设计要点：**
- **不可自发自收**：`send()` 入口即校验 `from_id == to_user_id`，防止异常数据
- **自动已读**：打开会话详情时调用 `mark_read(user_id, partner_id)`，未读数归零
- **会话聚合**：`list_conversations()` 按 partner 分组，提取最后一条消息、未读计数和最后通信时间
- **冗余容错**：获取对方昵称时 catch `NotFoundError`，返回 "unknown" 避免整个会话列表因数据异常而崩溃

#### 4.1.13 backend/app/services/forum_service.py - 论坛板块服务

板块服务管理论坛分区和板块的完整生命周期，含三层校验（不可自引用、类别一致性、有子板块不可删除）。

```python
from __future__ import annotations

from app.core.errors import ConflictError, ValidationError
from app.models.forum import ForumBoard, ForumBoardCategory
from app.repositories.forum_repository import InMemoryForumBoardRepository
from app.schemas.forum import (
    ForumBoardCreateRequest, ForumBoardUpdateRequest,
    ForumBoardView, ForumSectionView,
)


class ForumService:
    def __init__(self, board_repository: InMemoryForumBoardRepository) -> None:
        self.board_repository = board_repository

    def list_boards(
        self, category: ForumBoardCategory | None = None,
        market: str | None = None, active_only: bool | None = None,
    ) -> list[ForumBoardView]:
        """板块列表：支持 category/market 过滤，按 sort_order+时间排序"""
        boards = self.board_repository.list()
        filtered = [
            b for b in boards
            if (category is None or b.category == category)
            and (market is None or b.market == market)
            and (active_only is None or b.is_active == active_only)]
        filtered.sort(key=lambda b: (b.sort_order, b.created_at, b.name))
        return [self._to_view(b) for b in filtered]

    def get_section(self, category: ForumBoardCategory) -> ForumSectionView:
        """分区聚合：按category聚合下属板块，附加元信息"""
        board_views = self.list_boards(category=category)
        meta = {
            ForumBoardCategory.MARKET: ("市场讨论区", "按市场划分的投资讨论板块"),
            ForumBoardCategory.TOPIC: ("主题专区", "围绕投资方法与热点展开的主题讨论"),
            ForumBoardCategory.COMPANY_RESEARCH: ("公司研究专区", "按行业和个股进行深度研究"),
            ForumBoardCategory.QA: ("问答求助区", "新手提问和投资解惑集中区"),
        }
        title, description = meta[category]
        return ForumSectionView(
            category=category, title=title, description=description, boards=board_views)

    def list_sections(self) -> list[ForumSectionView]:
        """四大分区总览"""
        return [
            self.get_section(ForumBoardCategory.MARKET),
            self.get_section(ForumBoardCategory.TOPIC),
            self.get_section(ForumBoardCategory.COMPANY_RESEARCH),
            self.get_section(ForumBoardCategory.QA)]

    def create_board(self, request: ForumBoardCreateRequest) -> ForumBoardView:
        """创建板块：校验父板块存在性 + 类别一致性"""
        self._validate_parent(request.parent_id, request.category)
        board = ForumBoard(
            slug=request.slug, name=request.name,
            description=request.description, category=request.category,
            market=request.market, parent_id=request.parent_id,
            sort_order=request.sort_order, is_active=request.is_active)
        created = self.board_repository.create(board)
        return self._to_view(created)

    def delete_board(self, board_id: str) -> None:
        """删除板块：有子板块则拒绝删除，防止孤立数据"""
        board = self.board_repository.get(board_id)
        children = [b for b in self.board_repository.list()
                     if b.parent_id == board.id]
        if children:
            raise ConflictError("board has child boards, delete them first")
        self.board_repository.delete(board_id)

    def _validate_parent(
        self, parent_id: str | None, category: ForumBoardCategory,
        current_board_id: str | None = None,
    ) -> None:
        """父板块校验：不可自引用 + 类别一致性"""
        if parent_id is None:
            return
        parent = self.board_repository.get(parent_id)
        if current_board_id is not None and parent.id == current_board_id:
            raise ValidationError("parent board cannot be self")
        if parent.category != category:
            raise ValidationError("parent board category mismatch")

    def _to_view(self, board: ForumBoard) -> ForumBoardView:
        return ForumBoardView(
            id=board.id, slug=board.slug, name=board.name,
            description=board.description, category=board.category,
            market=board.market, parent_id=board.parent_id,
            sort_order=board.sort_order, is_active=board.is_active,
            created_at=board.created_at, updated_at=board.updated_at)
```

**板块服务关键业务规则：**
- **层级校验三重**：(1)不可自引用；(2)类别一致性；(3)级联删除保护
- **分区聚合**：`list_sections()` 将全部板块按四大分区聚合，附带中文标题和描述
- **排序策略**：sort_order 优先 → created_at → name 字母序

#### 4.1.14 backend/app/repositories/sqlite_backend.py - SQLite 持久化层

```python
from __future__ import annotations

import os, pickle, sqlite3, threading
from contextlib import contextmanager
from pathlib import Path

_INIT_LOCK = threading.Lock()
_INITIALIZED = False


def _resolve_db_path() -> Path:

DB_PATH = _resolve_db_path()


def _schema_sql() -> str:

def _ensure_initialized() -> None:
    """双重检查锁定（DCL）保证数据库文件+表结构单次初始化"""
    global _INITIALIZED
    if _INITIALIZED:
        return
    with _INIT_LOCK:
        if _INITIALIZED:
            return
        DB_PATH.parent.mkdir(parents=True, exist_ok=True)
        with sqlite3.connect(str(DB_PATH)) as conn:
            conn.executescript(_schema_sql())
            conn.commit()
        _INITIALIZED = True


@contextmanager
def get_connection():
    """线程安全连接上下文管理器"""
    _ensure_initialized()
    conn = sqlite3.connect(str(DB_PATH), check_same_thread=False)
    conn.row_factory = sqlite3.Row
    try:
        yield conn
    finally:
        conn.close()


def encode_payload(obj: object) -> sqlite3.Binary:
    return sqlite3.Binary(pickle.dumps(obj, protocol=pickle.HIGHEST_PROTOCOL))


def decode_payload(blob: bytes):
    return pickle.loads(blob)
```

**持久化层设计要点：**
- **payload BLOB 模式**：核心对象整体序列化为 pickle BLOB，同时冗余存储关键查询字段用于索引
- **联合主键**：`post_likes`/`post_favorites`/`follows` 使用联合主键防止重复操作
- **路径可配置**：`SOFT_RESIGN_SQLITE_PATH` 环境变量覆盖默认路径
- **线程安全**：`threading.Lock` + 双重检查锁定 + `check_same_thread=False`

#### 4.1.15 backend/app/repositories/content_repository.py - 内容数据仓储

```python
from __future__ import annotations
from datetime import UTC, datetime, timedelta

from app.core.errors import NotFoundError
from app.models.content import Comment, Post
from app.repositories.sqlite_backend import decode_payload, encode_payload, get_connection


class InMemoryContentRepository:
    """Backward-compatible class name; now backed by SQLite persistence."""

    def create_post(self, post: Post) -> Post:
        with get_connection() as conn:
            conn.execute(
                """INSERT INTO posts (id, board_id, author_id, created_at, payload)
                   VALUES (?, ?, ?, ?, ?)""",
                (post.id, post.board_id, post.author_id,
                 post.created_at.isoformat(), encode_payload(post)))
            conn.commit()
        return self.get_post(post.id)

    def get_post(self, post_id: str) -> Post:
        with get_connection() as conn:
            row = conn.execute(
                "SELECT payload FROM posts WHERE id = ?", (post_id,)).fetchone()
        if row is None:
            raise NotFoundError("post not found")
        return decode_payload(row["payload"])

    def list_posts(self) -> list[Post]:
        with get_connection() as conn:
            rows = conn.execute(
                "SELECT payload FROM posts ORDER BY created_at DESC").fetchall()
        return [decode_payload(r["payload"]) for r in rows]

    def create_comment(self, comment: Comment) -> Comment:
        self.get_post(comment.post_id)  # 校验帖子存在
        with get_connection() as conn:
            conn.execute(
                """INSERT INTO comments (id, post_id, author_id, parent_comment_id,
                   created_at, payload) VALUES (?, ?, ?, ?, ?, ?)""",
                (comment.id, comment.post_id, comment.author_id,
                 comment.parent_comment_id, comment.created_at.isoformat(),
                 encode_payload(comment)))
            conn.commit()
        return self.get_comment(comment.id)

    def list_comments(self, post_id: str) -> list[Comment]:
        self.get_post(post_id)
        with get_connection() as conn:
            rows = conn.execute(
                "SELECT payload FROM comments WHERE post_id = ? ORDER BY created_at ASC",
                (post_id,)).fetchall()
        return [decode_payload(r["payload"]) for r in rows]

    def set_like(self, post_id: str, user_id: str, enabled: bool) -> int:
        """点赞/取消点赞：INSERT OR IGNORE / DELETE 幂等设计"""
        self.get_post(post_id)
        with get_connection() as conn:
            if enabled:
                conn.execute(
                    """INSERT OR IGNORE INTO post_likes (post_id, user_id, created_at)
                       VALUES (?, ?, ?)""",
                    (post_id, user_id, datetime.now(UTC).isoformat()))
            else:
                conn.execute(
                    "DELETE FROM post_likes WHERE post_id = ? AND user_id = ?",
                    (post_id, user_id))
            conn.commit()
        return self.like_count(post_id)

    def set_favorite(self, post_id: str, user_id: str, enabled: bool) -> int:
        """收藏/取消收藏：INSERT OR IGNORE / DELETE 幂等设计"""
        self.get_post(post_id)
        with get_connection() as conn:
            if enabled:
                conn.execute(
                    """INSERT OR IGNORE INTO post_favorites (post_id, user_id,
                       created_at) VALUES (?, ?, ?)""",
                    (post_id, user_id, datetime.now(UTC).isoformat()))
            else:
                conn.execute(
                    "DELETE FROM post_favorites WHERE post_id = ? AND user_id = ?",
                    (post_id, user_id))
            conn.commit()
        return self.favorite_count(post_id)

    def is_liked(self, post_id: str, user_id: str) -> bool:
        with get_connection() as conn:
            row = conn.execute(
                "SELECT 1 FROM post_likes WHERE post_id=? AND user_id=?",
                (post_id, user_id)).fetchone()
        return row is not None

    def is_favorited(self, post_id: str, user_id: str) -> bool:
        with get_connection() as conn:
            row = conn.execute(
                "SELECT 1 FROM post_favorites WHERE post_id=? AND user_id=?",
                (post_id, user_id)).fetchone()
        return row is not None

    def like_count(self, post_id: str) -> int:
        with get_connection() as conn:
            row = conn.execute(
                "SELECT COUNT(1) AS c FROM post_likes WHERE post_id=?",
                (post_id,)).fetchone()
        return int(row["c"]) if row else 0

    def favorite_count(self, post_id: str) -> int:
        with get_connection() as conn:
            row = conn.execute(
                "SELECT COUNT(1) AS c FROM post_favorites WHERE post_id=?",
                (post_id,)).fetchone()
        return int(row["c"]) if row else 0

    def comment_count(self, post_id: str) -> int:
        with get_connection() as conn:
            row = conn.execute(
                "SELECT COUNT(1) AS c FROM comments WHERE post_id=?",
                (post_id,)).fetchone()
        return int(row["c"]) if row else 0

    def engagement_score(self, post_id: str) -> int:
        return (self.like_count(post_id) * 2
                + self.favorite_count(post_id) * 3
                + self.comment_count(post_id))

    def list_hot_posts(self, limit: int = 20, days: int = 7) -> list[Post]:
        """热榜：仅统计最近N天帖子，按 engagement_score 降序"""
        threshold = (datetime.now(UTC) - timedelta(days=days)).isoformat()
        with get_connection() as conn:
            rows = conn.execute(
                "SELECT payload FROM posts WHERE created_at >= ?",
                (threshold,)).fetchall()
        candidates = [decode_payload(r["payload"]) for r in rows]
        candidates.sort(
            key=lambda p: (self.engagement_score(p.id), p.created_at),
            reverse=True)
        return candidates[:limit]
```

**内容仓储关键设计：**
- **幂等互动**：`set_like`/`set_favorite` 使用 `INSERT OR IGNORE` + `DELETE`，重复操作不报错
- **互动状态查询**：`is_liked`/`is_favorited` 提供 viewer 维度互动状态
- **热榜时间窗口**：`list_hot_posts(days=7)` 仅统计最近 7 天，避免旧帖长期霸榜
- **engagement_score**：收藏(×3) > 点赞(×2) > 评论(×1)，体现不同互动形式的权重差异

#### 4.1.16 backend/app/services/content_filter.py - 内容审核过滤器

基于正则表达式的敏感词检测，涵盖 8 大违规类别，配合白名单避免误杀。

```python
"""Content filter — sensitive word filtering for posts and comments."""

from __future__ import annotations
import re

_SENSITIVE_WORDS: list[tuple[str, str]] = [
    (r"荐股|推.*票|带.*单", "荐股行为"),
    (r"涨停|跌停|拉升|砸盘", "操纵市场"),
    (r"加.*群|加.*微信|加.*QQ", "引流"),
    (r"赌博|博彩|赌场", "赌博"),
    (r"色情|黄色|成人", "色情"),
    (r"fake.*id|假.*证", "虚假证件"),
    (r"内幕|内部消息|庄家", "内幕交易"),
    (r"保证.*收益|稳赚|包赚", "虚假承诺"),
    (r"杠杆.*配资|场外配资", "非法配资"),
    (r"代客.*理财|代.*操盘", "违规代操"),
]

_WHITELIST = {"涨停板分析", "跌停板分析", "技术拉升", "内幕消息辟谣"}


class ContentFilterResult:
    def __init__(self, passed: bool, reason: str | None = None,
                 category: str | None = None):
        self.passed = passed
        self.reason = reason
        self.category = category


class ContentFilter:
    """Check content against sensitive word patterns."""

    def check(self, text: str) -> ContentFilterResult:
        if not text or not text.strip():
            return ContentFilterResult(passed=True)
        text_lower = text.lower()
        for pattern, category in _SENSITIVE_WORDS:
            match = re.search(pattern, text_lower)
            if match:
                matched_text = match.group(0)
                # 白名单优先：合法金融术语放行
                if any(wl in text for wl in _WHITELIST):
                    continue
                return ContentFilterResult(
                    passed=False,
                    reason=f'内容包含违规信息（匹配词: "{matched_text}"）',
                    category=category)
        return ContentFilterResult(passed=True)

    def get_violation_message(self, result: ContentFilterResult) -> str:
        if result.passed:
            return ""
        return f"[{result.category}] {result.reason}"


# 模块级单例
content_filter = ContentFilter()

from __future__ import annotations

import hashlib, hmac, os, secrets
from datetime import UTC, datetime, timedelta

PBKDF2_ALGO = "sha256"
PBKDF2_ITERATIONS = 240_000
TOKEN_TTL_HOURS = 24


def utc_now() -> datetime:
    return datetime.now(UTC)


def hash_password(password: str) -> str:
    """PBKDF2-HMAC-SHA256 哈希，格式: pbkdf2_sha256$240000$salt$digest"""
    salt = os.urandom(16)
    digest = hashlib.pbkdf2_hmac(
        PBKDF2_ALGO, password.encode("utf-8"), salt, PBKDF2_ITERATIONS)
    return f"pbkdf2_{PBKDF2_ALGO}${PBKDF2_ITERATIONS}${salt.hex()}${digest.hex()}"


def verify_password(password: str, encoded_hash: str) -> bool:
    """hmac.compare_digest 防时序攻击"""
    try:
        algo_label, iterations_str, salt_hex, digest_hex = encoded_hash.split("$")
    except ValueError:
        return False
    if not algo_label.startswith("pbkdf2_"):
        return False
    algo = algo_label.removeprefix("pbkdf2_")
    try:
        iterations = int(iterations_str)
        salt = bytes.fromhex(salt_hex)
        expected = bytes.fromhex(digest_hex)
    except (ValueError, TypeError):
        return False
    actual = hashlib.pbkdf2_hmac(algo, password.encode("utf-8"), salt, iterations)
    return hmac.compare_digest(actual, expected)


def generate_access_token() -> str:
    return secrets.token_urlsafe(32)


def token_expire_time() -> datetime:
    return utc_now() + timedelta(hours=TOKEN_TTL_HOURS)


def mask_id_number(id_number: str | None) -> str | None:
    """身份证脱敏：保留首尾2位，中间替换为*"""
    if not id_number:
        return None
    if len(id_number) <= 4:
        return "*" * len(id_number)
    return f"{id_number[:2]}{'*' * (len(id_number) - 4)}{id_number[-2:]}"

import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import api from '@/utils/api'

export const useAuthStore = defineStore('auth', () => {
  const token = ref(localStorage.getItem('access_token') || null)
  const userId = ref(localStorage.getItem('user_id') || null)

  const isLoggedIn = computed(() => !!token.value)

  function setSession(accessToken, id) {
    token.value = accessToken
    userId.value = id
    localStorage.setItem('access_token', accessToken)
    localStorage.setItem('user_id', id)
  }

  function clearSession() {
    token.value = null
    userId.value = null
    localStorage.removeItem('access_token')
    localStorage.removeItem('user_id')
  }

  async function register(payload) {
    const data = await api.post('/auth/register', payload)
    setSession(data.token.access_token, data.user_id)
    return data
  }

  async function login(payload) {
    const data = await api.post('/auth/login', payload)
    setSession(data.access_token, data.user_id)
    return data
  }

  function logout() {
    clearSession()
  }

  return { token, userId, isLoggedIn, setSession, clearSession,
           register, login, logout }
})

import axios from 'axios'

const api = axios.create({
  baseURL: '/api/v1',
  timeout: 15000,
})

// 请求拦截器：自动注入 Bearer Token
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截器：自动解包 data + 401 自动清理登录状态
api.interceptors.response.use(
  (response) => {
    const body = response.data
    if (body.code !== 0) {
      return Promise.reject(new Error(body.message))
    }
    return body.data  // 调用方直接拿业务数据，无需 .data.data
  },
  (error) => {
    const msg = error.response?.data?.message || error.message
    if (error.response?.status === 401) {
      localStorage.removeItem('access_token')
      localStorage.removeItem('user_id')
    }
    return Promise.reject(new Error(msg))
  },
)

export default api
```

## 6. 总结

### 6.1 项目成果

3. **自动化测试验证**：10 项 Python unittest 测试全部通过（10/10，2.680s），覆盖六大业务模块的正常流程和关键异常场景（不可自关注、不可自发自收、父评论归属校验、敏感词过滤等）。

4. **完善的文档体系**：交付 15 份文档，覆盖用户故事（28条，核心/重要/扩展三级优先级）、交互场景（8个 UC，含基本流程和异常流程）、架构与类设计、数据库设计（15+表，含 ER 图和索引策略）、接口设计（40+端点，含错误码）、前端 UI 设计（13页，含响应式方案）、实现说明、测试报告、AI 使用记录（12类交互）、分工记录、安装部署、用户使用说明、课程设计报告和评分点对照表。

5. **AI 辅助实践**：在需求分析、系统设计、编码实现、测试调试和文档撰写全流程中实践了 Human-In-the-Loop 的 AI 辅助开发模式。AI 负责初稿生成、代码骨架和结构化整理；人工负责需求边界确认、业务规则补充（联合主键/不可自关注/父评论归属校验）、接口路径对齐、测试环境配置和最终质量检查。

### 6.2 经验与收获

1. **分层架构的价值**：API-Service-Repository 三层分离使得业务规则、数据访问和接口处理各司其职。在增加"不可关注自己""不可给自己发私信""父评论归属校验"等业务约束时，仅需修改 Service 层，不影响接口和数据层。仓储抽象使得 SQLite -> MySQL 切换仅需替换仓储实现。

2. **AI 辅助的边界**：AI 在初稿生成、代码骨架和结构化整理方面效率很高（需求拆解、用户故事模板、接口骨架、测试草案），但业务规则（不可自关注、父评论归属校验、联合主键约束、敏感词白名单）和工程细节（接口路径 /api/v1 前缀统一、测试从 backend/ 目录运行、uv 缓存权限）必须由人工判断和修正。

3. **测试驱动信心**：10 项自动化测试覆盖核心业务闭环，使用 UUID 生成唯一测试数据保证可重复运行。每次代码修改后运行测试套件（2.680s），能快速验证功能完整性，避免回归问题。

4. **文档先行**：在设计阶段完成接口文档（40+端点）和数据库设计（15+表，联合主键，FULLTEXT 索引），编码阶段有明确的目标和约束，减少了返工。接口的 {code, message, data} 统一响应格式在设计阶段确定后，前端 API 封装和测试断言都基于此格式。

### 6.3 不足与改进方向

- 第三方服务集成：当前短信验证码、邮件服务、微信/微博 OAuth（SHA-256 stub）、人脸识别（始终返回 True）、KYC 身份核验均为预留接口。后续可接入阿里云短信、SendGrid 邮件、微信开放平台、腾讯云人脸识别等真实服务。
- 实时通信：私信和通知当前依赖客户端轮询，可升级为 WebSocket（Flask-SocketIO）实现实时双向通信和桌面通知。
- 前端测试：当前前端通过页面联调验证，可引入 Vitest（组件单元测试）+ Cypress/Playwright（端到端测试）补充前端自动化测试覆盖。
- 性能优化：可引入 Redis 缓存热榜（减少每次实时计算）、搜索联想（预计算热门股票代码）和会话列表。使用 gunicorn 多进程部署后端，Nginx 反向代理 + 静态资源 CDN。
- 群组功能：指导书中提到的投资主题群组（创建/加入/权限/群内讨论/资料分享）尚未实现，可在现有分层架构上扩展 Group/GroupMember/GroupPost 模型和服务。
- 附件审核：PDF/Excel 附件的上传、病毒扫描、内容审核流程可进一步完善，当前仅预留 image_urls 字段。
- 云上部署：当前以本地运行为主（Flask 开发服务器 + Vite 开发服务器），后续可使用 Docker 容器化，部署到云服务器（阿里云/腾讯云），配置 GitHub Actions CI/CD 自动测试和部署流水线。
